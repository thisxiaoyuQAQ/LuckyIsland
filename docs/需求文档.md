# LuckyIsland 需求文档（PRD）

> 灵动岛式 Windows 桌面助手 · 个人项目 lite 版
> 版本：v0.1 · 日期：2026-07-05
> 配套文档：[技术栈规划.md](./技术栈规划.md)

---

## 1. 项目背景与目标

### 1.1 背景
日常开发中，时间、天气、待办、股票、终端、AI 任务通知等信息散落在多个工具里，频繁切换窗口打断心流。Windows 缺少类似 macOS 灵动岛 / Dynamic Island 的「常驻轻量信息聚合层」。

### 1.2 目标
打造一个常驻屏幕顶部中央的灵动岛式面板，聚合高频信息与快捷操作，并通过 hook 接收 Claude/Codex 的任务完成通知，成为个人开发的「指挥中心」。

### 1.3 非目标（v1 不做）
- 多用户/账号体系
- 云同步（v1 仅本地）
- 移动端
- 完整插件市场（仅留接口，不做分发）
- 主动任务编排（不替用户跑 Claude/Codex，只接收通知与提供入口）

### 1.4 成功标准
- 常驻后台内存 < 60MB（含一个终端 tab）
- 冷启动到可交互 < 800ms
- 静态状态 CPU 占用 < 1%
- 连续运行 24h 无崩溃、无明显内存泄漏
- 个人日常使用一周后离不开

## 2. 用户画像与场景

**用户**：个人开发者（本人），Windows 11，多显示器，重度使用 Claude Code / Codex。

**典型场景**：
1. 编码时余光看灵动岛的时间/天气/下一个待办，不打断当前窗口
2. ALT+X 呼出，快速记一条待办 / 看一眼自选股 / 打开终端跑命令
3. Claude 在后台重构代码完成 → 灵动岛弹出「Claude 完成：重构 auth 模块」，无需切回终端查看
4. 一键快捷命令打开 codex / 切到项目目录 / 拉取代码
5. 专注模式：隐藏灵动岛，仅托盘存在；有通知时自动浮现

### 3.3 页面模型与切换

灵动岛采用「**单页画布**」：同一时刻只显示一个页面，每页只承载一种内容，通过快捷键或页签按钮切换。这是对原「多面板同屏并列」方案的调整——更贴合灵动岛 morph 隐喻，信息更聚焦，每页可用空间更大。

**页面清单**（默认顺序，均可独立开关）：

| 时间 | HH:MM，可加秒/日期/星期；启动默认页 | 本节 |

| 天气 | 当前温度+图标+未来 3 天；城市可配 + 多城市切换；10min 刷新；离线缓存最近一次 | 本节 |

## 6. UI / UX 规范

### 6.1 形态
- **隐藏态**：不可见，仅托盘
- **紧凑态**：顶部细条（高 ~40px），显示「当前页」的精简视图 + 页面指示点
- **展开态**：下拉面板（高 ~400px），显示「当前页」的完整内容 + 顶部页签/切换按钮

### 6.2 交互
- 鼠标进入岛区域 → 取消穿透 → 可交互
- 鼠标离开 + N 秒 → 自动收起为紧凑态（可配置）
- ALT+X → 切换 隐藏 ↔ 展开
- 点击紧凑态 → 展开
- ESC → 收起

### 6.3 视觉
- 圆角 16px（紧凑）/ 20px（展开）
- 毛玻璃 blur(20px)，背景 `rgba(..., 0.7)`
- 阴影：`0 8px 32px rgba(0,0,0,0.2)`
- 强调色用于：活动 tab、待办高亮、通知级别
- 字体：系统默认（Segoe UI Variable / Microsoft YaHei UI），终端用等宽（Cascadia Code / JetBrains Mono）
- 动画时长：状态过渡 200–280ms，缓动 `cubic-bezier(0.4, 0, 0.2, 1)`

### 6.4 可访问性
- 键盘可操作（Tab 导航、Enter 确认、ESC 收起）
- 对比度满足 WCAG AA
- 文字大小可调

## 7. 配置与 DIY

### 7.1 应用配置真源与早期未采用草案

当前 LuckyIsland 不读取应用 `config.toml`。设置统一存入 `app_data_dir()/data.db` 的 SQLite `settings(key,value)`，主要使用 `pages:*`、`general:*`、`terminal:*`、`window:*`、`wake:*`、`hotkeys:*`、`time:*` 以及 `notify:*`/`weather:*`/`stock:*`/`ai:*` 命名空间；设置 UI 修改后即时生效。

下列 TOML 仅保存为 2026-07-05 的**早期未采用草案**，不得作为当前配置接口或待补实现：

```toml
[window]
width = 720
height_compact = 40
height_expanded = 400
monitor = "primary"          # 或显示器名
state = "compact"            # hidden | compact | expanded
auto_collapse_seconds = 6

[shortcut]
toggle = "Alt+X"

[pages]
order = ["time", "calendar", "weather", "stock", "todo", "terminal"]
enabled = { weather = true, stock = true }
default = "time"
```

### 7.2 CLI 工具 `lucky-notify`
```
lucky-notify --title "Claude 完成" --body "重构 auth 模块" --source claude --level success
```
- CLI 使用 `reqwest::blocking::Client` 直接 POST 本地 HTTP 端点，并显式 `no_proxy`
- token 依次来自 `--token`、`LUCKY_TOKEN`、SQLite `notify:http_token`
- 主进程未运行或请求失败时直接报错；当前没有命名管道或离线队列
- 安装时加入 PATH

### 7.3 Claude Code / Codex hook 模板
- **Claude Code**：在 `~/.claude/settings.json` 配 `Stop` hook 调用 `lucky-notify`
- **Codex**：在 `~/.codex/config.toml` 配 completion 钩子
- 文档提供一键脚本，安装即用
