# RUN-10C-06 KWS / ASR / TTS / 模型下载 / 安装态

> 来源：AUD-RUNTIME-06
> 通过条件：下载可恢复、转写/TTS 正常、DLL 齐全

## 前置

- Release 安装且语音模型未下载（或先删除 `%APPDATA%\com.luckyisland\models\`）
- 麦克风可用且 Windows 隐私设置允许 LuckyIsland 使用
- 网络可访问 GitHub Release（必要时备代理）

## 步骤与预期

| # | 操作 | 预期 |
|---|---|---|
| 1 | 设置 → 语音 → 下载唤醒模型 | 进度条可见；下载完成后状态变「已就绪」；模型目录含 `kws/` 子目录 |
| 2 | 下载问答模型（ASR） | 同上；含 `asr/` 子目录 |
| 3 | 下载中断网 30s 后恢复 | 下载能断点续传或明确失败重试，不会卡 100% 不动；日志含明确错误行 |
| 4 | 开启「语音唤醒」开关 | 后端启动监听线程；设置面板状态切到「监听中」；AI 面板可隐藏 |
| 5 | 对麦克风说默认唤醒词「小岛小岛」 | 1s 内听到 TTS 播报「主人我在」；AI 面板弹出并进入聆听态 |
| 6 | 唤醒后说一个简单问题（如「今天天气怎么样」） | ASR 转写为中文文本并自动发送给当前 provider；AI 回复正常 |
| 7 | 自定义唤醒词（设置面板改为新词） | 自动热重载；新词生效，旧词不再触发 |
| 8 | 自定义唤醒应答（如「在的主人」） | 唤醒后 TTS 播报新应答；无乱码 |
| 9 | 检查 DLL | 安装目录含 sherpa-onnx / onnxruntime 共 4 个 DLL；版本与 `docs/技术栈规划.md` 一致 |
| 10 | 步骤 4–8 连续做 3 轮 | 每轮都正常，无内存/句柄异常增长 |
