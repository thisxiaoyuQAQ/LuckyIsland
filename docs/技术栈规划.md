# LuckyIsland 技术栈规划

> 灵动岛式 Windows 桌面助手 · 个人项目 lite 版
> 版本：v0.1 · 日期：2026-07-05

---

## 0. 决策摘要（TL;DR）

| 维度 | 选型 | 理由 |
|---|---|---|
| 应用框架 | **Tauri 2.x** | 已定（路径 `Tauri/LuckyIsland`）。Rust 后端，WebView2 渲染，内存 ~15MB，启动 <500ms |
| 前端框架 | **React 19 + TypeScript** | 生态成熟，xterm.js / Framer Motion / shadcn 适配最佳 |
| 样式 | **Tailwind CSS v4 + shadcn/ui** | 原子化 + 可复制组件，DIY 友好，主题切换原生支持 |
| 动画 | **Motion (Framer Motion)** | 灵动岛展开/收缩/切换的核心体验 |
| 状态 | **Zustand + persist** | 轻量，TS 友好，持久化简单 |
| 终端 | **xterm.js + portable-pty (Rust)** | 唯一可行的「面板内嵌终端」方案 |
| 配置/数据 | **SQLite (rusqlite) + TOML** | 结构化数据 + 人类可读配置 |
| 通知桥 | **CLI helper + 本地 HTTP** | 兼容 Claude Code / Codex 的 hook 机制 |

> 备选前端：**Svelte 5**（运行时更小、动画更顺手，但生态/集成度不如 React，DIY 资源少）。若极度追求体积可切换，初期不建议。

## 1. 整体架构

```
┌──────────────────────────────────────────────────────────┐
│                   LuckyIsland (Tauri 2)                  │
│                                                          │
│  ┌──────────────────── Frontend (WebView2) ───────────┐  │
│  │  React 19 + TS + Tailwind + shadcn/ui + Motion     │  │
│  │  ┌────────────────────────────────────────────────┐  │  │
│  │  │ 单页画布：同一时刻只渲染一个页面              │  │  │
│  │  │ 页面 = 时间 / 日历 / 天气 / 股票 / 待办 /     │  │  │
│  │  │        终端 / 通知                            │  │  │
│  │  │ 切换 = Alt+1~9直达 · 页签按钮 · Alt+←/→循环   │  │  │
│  │  └────────────────────────────────────────────────┘  │  │
│  │  Zustand store ←→ Tauri IPC (event/invoke)         │  │
│  └────────────────────────┬───────────────────────────┘  │
│                           │                              │
│  ┌──────────────────── Backend (Rust) ─────────────────┐  │
│  │  core: window / config / theme / tray / shortcut    │  │
│  │  data: weather / stock / calendar / todo            │  │
│  │  terminal: portable-pty ↔ xterm bridge              │  │
│  │  notify:  HTTP server + IPC + Claude/Codex hooks    │  │
│  │  storage: SQLite + TOML                             │  │
│  └─────────────────────────────────────────────────────┘  │
│                                                          │
│  ┌─────── OS 接口 ───────┐  ┌─────── 外部 ────────┐      │
│  │ Win32 / WebView2 /   │  │ 天气API 股票API     │      │
│  │ 全局热键 / 托盘 / 通知│  │ Claude/Codex hooks  │      │
│  └──────────────────────┘  └────────────────────┘      │
└──────────────────────────────────────────────────────────┘
```

**核心原则**
- 前端只负责「展示 + 交互」，所有 IO/系统调用走 Rust
- **单页画布**：同一时刻只渲染一个页面，按键/页签按钮切换，贴合灵动岛 morph 隐喻
- 模块边界清晰：每个页面是独立组件 + 独立后端 service，可单独开关
- 配置驱动一切：页面顺序、开关、尺寸、主题、快捷键全部可配置

## 2. 前端技术栈

### 2.1 框架与语言
- **React 19** + **TypeScript 5.x**（strict 模式）
- 构建器：**Vite 6**（Tauri 默认模板）
- 路由：无需（单窗口应用，面板内 tab 切换）

### 2.2 UI 与样式
| 库 | 用途 |
|---|---|
| Tailwind CSS v4 | 原子化样式，`dark:` 原生支持主题 |
| shadcn/ui | 基础组件（Button / Dialog / Dropdown / Tooltip / Switch），复制即用、完全可控 |
| Radix UI | shadcn 底层，无障碍 + 键盘导航 |
| lucide-react | 图标 |
| clsx + tailwind-merge | 类名合并 |
| Motion (Framer Motion) | 灵动岛变形、页面切换(morph)、通知进入动画 |
| @xterm/xterm + @xterm/addon-fit + addon-webgl + addon-web-links | 终端渲染 |

### 2.3 状态与数据
- **Zustand**：全局状态（窗口状态、当前面板、主题、配置缓存）
- **zustand/middleware persist**：缓存到 localStorage（启动期默认值，真实配置从 Rust 同步）
- **TanStack Query**：仅用于股票/天气等异步数据（缓存、轮询、去重）

### 2.4 主题系统
- CSS 变量定义 `--bg / --fg / --accent / --muted ...`
- `data-theme="light|dark"` 在 `<html>` 上切换
- Tailwind v4 `@theme` 配置色板
- 跟随系统：`prefers-color-scheme` + Tauri 主题事件
- 自定义强调色：用户配置 hue，HSL 派生整套色阶

## 4. 关键技术难点与方案

### 4.1 透明无边框置顶窗口（灵动岛外壳）
**Tauri 配置**（`tauri.conf.json`）：
```jsonc
"windows": [{
  "label": "island",
  "decorations": false,
  "transparent": true,
  "alwaysOnTop": true,
  "skipTaskbar": true,
  "resizable": false,
  "shadow": false,            // 阴影用 CSS box-shadow 自绘，更可控
  "visible": false,           // 启动后由 Rust 控制显示
  "width": 720, "height": 80  // 默认尺寸，可配置
}]
```
- 圆角 + 毛玻璃：`backdrop-filter: blur(20px)` + `border-radius` + `box-shadow`
- Windows 透明窗口需启用 WebView2 的透明合成，性能可接受（实测 <2% CPU 静态）
- DPI 缩放：监听 `scale-change` 事件，前端用 `device-pixel-ratio` 适配

### 4.2 点击穿透（岛外区域不挡视线）
- `appWindow.setIgnoreCursorEvents(true)` 让整窗穿透
- 前端用 `mouseover` 检测鼠标是否进入岛区域，进入时调用 `setIgnoreCursorEvents(false)`
- 隐藏态下完全穿透；呼出态下岛内可交互
- **坑**：穿透模式下窗口收不到 mouseleave，需用全局鼠标轮询或定时器兜底

### 4.3 多显示器选择
- `availableMonitors()` 枚举，记录 `name()` 作为稳定标识
- 配置存 `monitor: "DELL-xxxx"`，启动时匹配；找不到则回退主屏
- 定位：`window.set_position(...)` 用显示器 `position() + size()` 计算顶部居中
- 提供「跟随主屏」选项，热插拔时自动迁移

### 4.4 全局快捷键
- `tauri-plugin-global-shortcut` 注册 `Alt+X`（切换显示）
- 用户可配置：`Ctrl+Space`、`Win+...`（注意 Win 键组合部分被系统占用）
- 支持「按住才显示 / 切换显示」两种模式
- 冲突检测：注册失败时提示用户

### 4.5 嵌入式终端（核心难点）
**为什么不能直接嵌 wt**：Windows Terminal 是独立终端模拟器，无法把它的渲染区域嵌入到 WebView 内。所以方案是「在面板内用 xterm.js 实现 WT 风格终端，spawn 用户配置的 shell」。

**架构**：
```
xterm.js  ──stdin──▶  Tauri invoke/event  ──▶  portable-pty write
xterm.js  ◀──stdout──  Tauri event push    ◀──  portable-pty read (tokio task)
```
- 默认 shell：读取 `COMSPEC` / `SHELL`，默认 PowerShell 7 → Windows PowerShell → cmd
- 渲染器：`addon-webgl`（性能），降级 `addon-canvas`
- 主题：读取 Windows Terminal 的配色 JSON，内置几套（One Half Dark、Campbell）
- **快捷命令**：配置文件定义 `{ name, command, cwd?, env? }`，点击即在新 tab 执行
  - 例：`{ "codex": { command: "codex", cwd: "E:\\Code" } }`
- 「呼出 wt」语义：面板终端默认全功能；额外提供「在外部 Windows Terminal 打开」按钮（spawn `wt.exe -d <cwd>`）
- 尺寸：`addon-fit` 自动适配，resize 时通过 pty resize 通知子进程
- 多实例：每个终端 tab 一个 PtyPair，关闭时 kill 子进程

### 4.6 实时数据源
| 数据 | 源 | 频率 | 备注 |
|---|---|---|---|
| A 股行情 | 腾讯 `qt.gtimg.cn` / 新浪 `hq.sinajs.cn` | 3–5s 轮询 | 免费、无需 key、有反爬，需带 Referer |
| 美股/加密 | Yahoo Finance `query1.finance.yahoo.com` | 10s | 需处理 cookie，不稳时降级 |
| 天气 | 和风天气 QWeather（开发版免费） | 10min | 中国城市友好，含空气质量 |
| 日历/农历 | 本地计算（`lunisolar` crate） | 启动一次 | 无网络依赖 |

- 所有 HTTP 走 Rust `reqwest`，避免前端 CORS 和密钥泄露
- 股票开收盘判断：仅在交易时段高频轮询，否则降频到 30s
- 失败重试 + 指数退避 + 离线最近一次缓存

### 4.7 Codex / Claude 完成通知（特色功能）
**目标**：Codex/Claude 完成任务时灵动岛弹通知卡片。

**双通道设计**：
1. **CLI helper（推荐）**：随主程序安装 `lucky-notify.exe`，通过本地命名管道把消息推给运行中的主进程
   - 用法：`lucky-notify --title "Claude 完成" --body "重构了 auth 模块" --level success`
   - Claude Code 在 `Stop` hook 里调用：`lucky-notify "任务完成: $CLAUDE_TASK"`
   - Codex 在其 completion hook / `~/.codex/config.toml` 里调用
2. **本地 HTTP 端点（备用）**：主进程起 `127.0.0.1:9753/notify`（POST JSON），适合无法 spawn 进程的场景
   - 带简单 token 鉴权，防止恶意网页 POST

**通知呈现**：
- 灵动岛展开 → 滑入通知卡片（标题/正文/来源/时间）
- 同步系统通知（可关闭）
- 可选声音 + 强调色闪烁
- 通知历史存 SQLite，可回看

### 4.8 主题与 DIY
- 亮/暗：CSS 变量 + Tailwind `dark:` + 跟随系统
- 强调色：HSL 派生，用户调一个 hue
- **DIY 三层**：
  1. 配置层（近期）：`config.toml` 控制面板开关、顺序、尺寸、快捷键、自选股、快捷命令
  2. 主题层（中期）：导出/导入主题 JSON
  3. 插件层（远期）：定义 Panel 接口，用户写 JS 模块（沙箱化，受限 API）注入自定义面板

## 5. 数据与存储

| 数据 | 存储 | 路径 |
|---|---|---|
| 应用配置 | TOML | `%APPDATA%\LuckyIsland\config.toml` |

## 9. 风险与对策

| 风险 | 影响 | 对策 |
|---|---|---|
| 透明窗口在部分显卡驱动上掉帧 | 体验 | 提供「关闭透明/关闭毛玻璃」降级开关 |
| ConPTY 在低版本 Win10 不可用 | 兼容 | 最低支持 Win10 1903+；检测后禁用终端面板 |
| 股票 API 被反爬/限流 | 数据不稳 | 多源切换（腾讯↔新浪）+ 缓存 + 指数退避 |
| xterm.js 大输出卡顿 | 终端体验 | webgl 渲染 + scrollback 限制 + 流控 |
| 全局热键冲突 | 功能不可用 | 注册失败提示 + 可配置改键 |
| WebView2 内存随时间增长 | 长期运行 | 提供重启托盘项 + 终端 tab 主动回收 |
| 通知 hook 用户不会配 | 功能落空 | 提供 `lucky-notify` 安装即用脚本 + Claude/Codex 配置模板 |

## 10. 备选方案记录

- **前端 Svelte 5**：体积更小，但 xterm/Framer 生态弱，DIY 资源少 → 暂不选，留作 v2 评估
- **Electron**：生态强但内存 100MB+，违背「高性能」 → 否决
- **终端改为外弹 wt.exe**：实现简单，但不符合「面板内显示」诉求 → 仅作辅助按钮
- **通知用文件监听**：极简但不可靠、延迟大 → 仅作降级
- **配置用 YAML**：与 TOML 等价，但 Tauri/Rust 社区 TOML 更主流 → 选 TOML
