# LuckyIsland 时间页可自定义组件设计

> 日期：2026-07-11
> 状态：已批准，待实施
> 范围：重构时间页展开态为可配置的九宫格组件画布；新增一言、程序员历史上的今天、今日运势、电子木鱼、今日心情；新增独立的“时间组件”和“时间外观”设置面板

## 1. 背景与目标

当前时间页展开态只显示时间、日期和星期，未充分利用灵动岛 `720×400` 窗口中的展开区域。本次改造把时间页变成低干扰、可配置的时间仪表盘：

- 时间块与组件均可放置到九宫格区域；
- 用户可开关组件、指定区域并调整同区域内顺序；
- 时间、日期和星期可独立配置显示、纯色或双色渐变；
- 收起态保持现有单行时钟，不引入额外信息；
- 网络组件具备超时、缓存和离线降级；
- 运势、木鱼和心情全部本地运行，不上传用户数据；
- 所有参考项目只用于理解功能，不直接复制无明确许可证的源码或素材。

## 2. 非目标

首版不实现：

- 像素级自由拖拽、任意缩放或组件相互覆盖；
- 用户编写 CSS、多色渐变、文字描边、发光或自定义字体；
- 电子木鱼在线排行榜、昵称、账号、上传功德或反作弊；
- 自选本地音频文件；
- 心情备注、趋势图、月历或云同步；
- 运势的星座、生肖或个人资料推算；
- 第三方组件插件 SDK 或运行时动态加载组件。

## 3. 现有约束

- `island` 窗口紧凑态为 `720×80`，展开态为 `720×400`。
- React 外壳最大宽度为 700px，展开内容位于 56px 顶栏下方，根节点禁止页面级溢出。
- 页面使用静态注册表，时间页遵循 `({ compact }: { compact: boolean })` 契约。
- 设置使用 SQLite 字符串 KV，通过 `settings://changed` 实时通知主窗口。
- 远程数据应由 Rust `reqwest` 请求；直接从 WebView 请求会受 CORS 影响，也不符合现有数据页架构。
- 现有项目没有浏览器音频播放模式，木鱼应使用随应用打包且许可清晰的短音频。

## 4. 选定方案

采用“时间画布 + 九宫格区域插槽”。九宫格用于表达相对位置和稳定排序，不代表像素级自由画布：

```text
┌──────────┬──────────┬──────────┐
│   左上   │   上方   │   右上   │
├──────────┼──────────┼──────────┤
│   左侧   │   中央   │   右侧   │
├──────────┼──────────┼──────────┤
│   左下   │   下方   │   右下   │
└──────────┴──────────┴──────────┘
```

### 4.1 时间块

- 默认位于中央，可移动到任意一个区域。
- 时间块独占所在区域，组件不能与其重叠。
- 时间移动到已占用区域时执行区域交换：目标区域的全部组件按原顺序移动到时间原区域，时间进入目标区域。
- 时间块包含时间、日期、星期；隐藏其中一项不会改变时间块的区域归属。

### 4.2 组件区域与顺序

- 每个启用组件必须属于一个区域。
- 多个组件可位于同一区域，并按用户顺序纵向排列。
- 同一区域空间不足时，组件切换为摘要形态；仍不足时，该区域内部滚动，不让整个应用窗口滚动。
- 上方、下方及中间横向区域可使用较宽形态；角落和左右区域使用紧凑形态。
- 禁用组件时保留其区域和顺序配置，再次启用后恢复。

### 4.3 收起态

收起态继续只显示单行时钟，不渲染五个扩展组件，不显示摘要，不改变当前顶栏控制区宽度。

## 5. 组件注册与边界

新增时间页内部组件注册表。每个组件有稳定 ID、显示名称、默认区域、默认顺序、渲染能力和设置定义。首版固定注册以下 ID：

```text
saying
programmer_history
fortune
wooden_fish
mood
```

注册表只管理内置组件，不开放第三方代码注入。`TimePage` 负责读取布局并组合组件；每个组件独立负责自己的数据、交互和紧凑/摘要渲染，避免把网络请求、计数和样式逻辑堆进一个文件。

## 6. 组件详细设计

### 6.1 一言

数据源：UAPI `GET https://uapis.cn/api/v1/saying`，响应核心字段为 `text`。

行为：

- 默认在进入时间页后按需加载一次；
- 卡片最多显示两行，超出部分省略；
- 用户可点击卡片或刷新按钮换一句；
- 设置项控制“进入页面时刷新”和“允许点击换一句”；
- 每次成功响应覆盖最后成功缓存；
- 请求失败时显示缓存，并标记为缓存内容；
- 没有缓存时显示随应用提供的原创短句，卡片不留空。

不设置自动轮询，避免内容跳动和无意义请求。

### 6.2 程序员历史上的今天

数据源：UAPI `GET https://uapis.cn/api/v1/history/programmer/today`。响应包含 `date` 与 `events`；事件包含 `year`、`title`、`description`、`category`、`tags`、`importance`、`source` 等字段。

行为：

- 按本地日期每天首次需要时请求一次；
- 小卡片显示年份、事件标题和可选分类标签；
- 多事件通过左右控制切换；用户正在阅读时不自动切走；
- 可选自动轮换，默认关闭；
- 点击事件打开时间页内部轻量详情层，展示完整描述和标签；
- 缓存按 `MM-DD` 保存，接口失败时可显示同月同日缓存并标记“缓存”；
- 无缓存时显示明确的加载失败状态和重试入口，不伪造历史事件。

### 6.3 今日运势

参考 `luodeb/DailyFortunePlugin` 的“运势等级、签语、星级、幸运数字、幸运颜色”概念，使用全新 TypeScript/Rust 实现和原创内容库。

显示内容：

- 运势等级；
- 原创运势签语；
- 星级；
- 幸运数字；
- 幸运颜色名称、色值和颜色圆点；
- 小型“娱乐内容”提示。

行为：

- 每天首次打开自动生成一签；
- 用户可反复点击“再抽一次”，每次覆盖当天当前结果；
- 当前结果和日期保存在本地，应用重启后保持；
- 本地日期变化后首次打开自动生成新结果；
- 重抽使用短暂翻牌或流光动画，设置中可关闭动画；
- 随机逻辑不接受姓名、生日、位置等个人信息。

参考仓库未声明许可证，因此不复制其 Python 源码、整批配置文案或颜色表。

### 6.4 电子木鱼

参考 `mrnantang/muyu` 的点击木鱼、功德计数、缩放和浮字交互，按 LuckyIsland 视觉和技术栈重新实现。

显示内容：

- 原创绘制或许可明确的木鱼图形；
- 今日功德；
- 累计功德；
- 声音状态。

敲击流程：

1. 木鱼短暂缩放；
2. 播放一次打包音效；
3. 浮出“功德 +1”；
4. 今日与累计功德立即在 React 状态中各加一；
5. 用户停止敲击后防抖批量写入 SQLite。

规则：

- 本地日期变化后今日功德清零，累计功德保留；
- 快速连续点击不得丢计数；
- 对并发音效数量设上限，避免叠音失控；
- 设置支持声音开关、音量、动画开关和疯狂星期四文案开关；
- 不提供本地音频选择；
- 不联网、不上传数据、不提供排行榜。

#### 疯狂星期四彩蛋

- 仅在本地星期四启用；
- 今日功德首次达到 10、50、100，之后每增加 100 时显示一条随机原创短文案；
- 当日已经触发的里程碑不因应用重启而重复显示；
- 文案只影响展示，不改变功德增量和累计规则；
- 默认开启，可在设置中关闭；
- 参考用户提供文章所代表的网络文案风格，不逐段复制文章内容。

示例语气：

- “木鱼一敲，烦恼清零。今天周四，V 我 50 看看实力。”
- “功德已经到账，炸鸡仍在路上。”
- “施主今日与佛有缘，也与疯狂星期四有缘。”

音效、图片和代码不得取自未声明许可证的参考仓库；新增素材必须原创或具有允许应用分发的明确许可证，并记录来源。

### 6.5 今日心情

固定五档：

```text
很棒 / 开心 / 平静 / 疲惫 / 低落
```

行为：

- 用户每天选择一个心情，当天允许修改；
- 卡片显示今日心情和当前连续记录天数；
- 连续天数基于本地日期记录计算；漏记一个完整自然日后中断；
- 修改当天心情不重复增加连续天数；
- 设置中可隐藏连续天数；
- 数据仅存本地，不上传。

首版不显示文字备注、历史趋势和月历。

## 7. 独立设置面板

“页面管理”仍只负责启用时间页以及页面级排序。时间页内部设置拆成两个独立面板、独立组件文件和独立设置分组。

### 7.1 时间组件

负责：

- 五个组件的启用开关；
- 每个组件的九宫格区域；
- 同一区域内拖动排序；
- 简化九宫格实时预览；
- 每个组件的专属选项。

组件专属设置：

| 组件 | 设置 |
|---|---|
| 一言 | 进入页面时刷新、允许点击换一句 |
| 程序员历史 | 显示分类、自动轮换事件 |
| 今日运势 | 动画开关 |
| 电子木鱼 | 声音、音量、动画、疯狂星期四文案 |
| 今日心情 | 显示连续天数 |

### 7.2 时间外观

负责时间块位置和文字表现：

- 时间块九宫格位置；
- 时间、日期、星期分别显示或隐藏；
- 12/24 小时制；
- 是否显示秒；
- 时间字号档位；
- 时间字重：常规或加粗；
- 时间、日期、星期分别选择纯色或双色渐变；
- 渐变方向：水平、垂直、左上至右下、右上至左下；
- 十六进制颜色输入和可视化取色器；
- 预设主题；
- 单项恢复默认和全部恢复默认。

渐变通过 CSS 文本渐变实现。非法色值在设置层拒绝保存；运行时解析到非法旧值时使用默认色。

### 7.3 实时同步

设置通过现有 `setting_set_and_emit` 保存，并由时间页订阅 `settings://changed`。成功保存后立即刷新布局或样式，无需重启。写入失败时设置面板恢复旧值并显示错误。

## 8. 设置与本地数据模型

延续 SQLite 字符串 KV。布局和复杂配置以带版本号的 JSON 保存，解析时执行字段白名单、默认值补全和非法值回退。

建议设置分组：

```text
time:layout
 time:appearance
 time:widget:saying
 time:widget:programmer_history
 time:widget:fortune
 time:widget:wooden_fish
 time:widget:mood
```

其中 `time:layout` 至少包含：

```text
version
clockRegion
widgets[]: { id, enabled, region, order }
```

持久数据与展示设置分离：

```text
time:data:saying:last
 time:data:programmer_history:<MM-DD>
 time:data:fortune
 time:data:wooden_fish
 time:data:mood:<YYYY-MM-DD>
```

要求：

- 未知组件 ID 被忽略；缺失的已知组件按默认顺序补入；
- 重复组件 ID 去重；
- 非法区域回退到该组件默认区域；
- 同区域顺序归一化为连续整数；
- 时间目标区域冲突时按区域交换规则修复，而不是丢弃组件；
- JSON 版本升级必须集中在解析函数中，组件本身不处理迁移。

## 9. 配置导入导出边界

纳入普通配置导入导出：

- 时间块位置；
- 组件启用、区域和顺序；
- 时间、日期、星期的显示与外观；
- 五个组件的专属设置。

不纳入普通配置导入导出：

- 一言与历史缓存；
- 当日运势结果；
- 今日及累计功德；
- 心情历史和连续天数。

这些属于运行数据或个人记录，导入外观配置时不得覆盖。Rust 的 portable settings 白名单需加入 `time:` 设置键，但明确排除 `time:data:`。

## 10. 数据流

### 10.1 网络组件

```text
React 时间组件
  → Tauri command
  → Rust 数据模块
  → 共享 reqwest Client + 明确超时
  → UAPI
  → HTTP 状态检查 + 类型校验
  → SQLite 缓存
  → 新数据或带 stale 标记的缓存
```

Rust 为一言和程序员历史提供独立类型与命令。每次请求必须区分：请求失败、非成功状态、响应读取失败和 JSON/字段校验失败。

### 10.2 本地交互组件

- 运势：生成后立即保存当前结果；重抽覆盖当天结果。
- 木鱼：内存即时计数，停止敲击后防抖写入；组件卸载和窗口关闭路径尽力刷新待写数据。
- 心情：按日期 upsert 当天记录；连续天数从本地记录计算，不把派生值作为唯一事实来源。

### 10.3 日期边界

所有“今日”逻辑使用用户本地自然日。组件挂载时、窗口重新获得可见状态时以及跨午夜定时检查时重新计算日期，避免应用长时间运行后仍保留前一天状态。系统时间回拨时不删除历史数据，只按当前本地日期读取对应记录。

## 11. 异常与降级

- UAPI 超时或不可用：返回同类型缓存并标记 stale；无缓存时显示错误和重试。
- 一言无缓存：使用内置原创短句。
- 程序员历史无缓存：不生成虚假事件。
- 响应结构改变：视为解析失败，不把部分错误数据写入缓存。
- 音频播放失败：仍增加功德，仅在组件内显示短暂的声音不可用提示。
- SQLite 写入失败：保留当前会话内存状态并显示非阻塞提示；后续操作再次尝试保存。
- 布局配置损坏：解析器恢复默认并保留应用可用性。
- 所有外部文案作为纯文本渲染，不使用 `dangerouslySetInnerHTML`。

## 12. 可访问性与动效

- 所有可点击卡片和图标使用按钮语义、键盘焦点和可读标签；
- 颜色不是运势、心情或状态的唯一表达方式；
- 渐变文字保证有可读的默认回退色；
- 遵循 `prefers-reduced-motion`：关闭翻牌、流光、木鱼缩放和浮字位移动画，但保留状态更新；
- 木鱼音量默认温和，声音可一键关闭；
- 详情层支持 Escape 关闭并恢复焦点。

13. 心情当天可修改，连续天数计算正确；
14. 设置变更实时同步；
15. 配置导入导出恢复布局和外观，但不覆盖功德、心情等运行数据。
