# LuckyIsland 插件专项阶段 0：威胁模型与产品规则设计

> 日期：2026-07-14  
> 状态：阶段 0 设计已获用户最终批准；独立安全复核 Pass；阶段 0 已完成  
> 上游路线图：[2026-07-13-plugin-market-roadmap.md](./2026-07-13-plugin-market-roadmap.md)  
> 适用平台：Windows 10/11 x64  
> 本阶段性质：安全与产品规则设计，不编写插件运行时生产代码

## 1. 摘要

LuckyIsland 将建设公开第三方插件生态，并最终把问答和语音迁移为可独立安装、更新和卸载的官方插件。插件系统会执行来自外部发布者的代码，访问网络、用户文件、麦克风、剪贴板、子进程和宿主 UI，因此不能把“独立进程”“市场审核”或“用户点过同意”当作完整安全边界。

阶段 0 确立三维正交策略：

1. **安装来源**决定包能否进入系统：市场审核包、发布者签名侧载包、未签名开发包；
2. **执行载体**决定必须建立的运行环境：WASM/WASI、受限原生 Host、隔离 Web UI；
3. **能力风险**决定授权和运行限制：低风险、中风险、高风险 capability。

安装、启动和每次能力调用分别裁决。任何一层失败均默认拒绝。市场审核、“允许不安全插件”和开发者模式都不是万能通行证。

第三方二进制永远不得动态加载进 LuckyIsland 主进程。所有第三方原生插件都必须在宿主成功建立并验证 Windows 受限运行环境后启动；隔离失败时拒绝运行，不允许用户降级为普通当前用户权限运行。

## 2. 背景与现状约束

LuckyIsland 当前是 Tauri 2 + React 19 的 Windows 桌面应用。Rust 主进程集中注册 Tauri command 和 managed state；SQLite settings KV 是跨窗口持久状态真源；热键、通知、窗口政策、AI Provider 和语音生命周期仍属于宿主内部模块。现有 AI 与语音包含 CLI 子进程、网络、密钥、对话历史、麦克风、sherpa-onnx DLL、模型和 Windows SAPI 等高风险资源。

现有架构可作为后续迁移输入，但不构成插件安全边界：

- 插件不能获得宿主全局 Tauri `invoke`；
- 插件不能直接读取宿主 SQLite、通知 Token 或其他模块状态；
- 插件不能直接裁决窗口 show/hide/focus/click-through；
- 现有 AI/voice 代码在阶段 0 不移动、不删除、不改造；
- 现有 sherpa-onnx 依赖和 DLL 在阶段 0 保持不变。

阶段 0 单独立项。阶段 1 必须在本规格完成安全复核和用户最终批准后，另行设计与规划。

## 3. 目标与非目标

### 3.1 目标

本阶段产出可审查、可推演、可供后续原型验证的安全和产品契约：

- 受保护资产、攻击者能力和信任假设；
- LuckyIsland 主进程、Plugin Bridge、插件执行域和外部资源的信任边界；
- 安装来源、执行载体和能力风险的正交策略；
- Windows 10/11 x64 原生插件最低隔离承诺；
- 安装、启动、调用、更新、回滚、禁用、撤销和卸载规则；
- 两级侧载、自动复锁和防误用规则；
- 发布者身份、签名、审核、下架、撤销和申诉规则；
- 离线运行、隐私、遥测、本地审计和性能预算原则；
- 阶段 1 的允许范围、禁止范围和进入门禁。

### 3.2 非目标

阶段 0 不进行下列工作：

- 不新增插件源码目录、Plugin Host、manifest parser 或市场 API；
- 不实现安装、启用、禁用、更新、回滚或卸载入口；
- 不选择或引入 WASI 运行时、RPC 框架或 Windows 隔离封装库；
- 不实现 AppContainer、受限 Token、Job Object、网络代理或 IPC；
- 不迁移或卸载 AI、语音及任何现有模块；
- 不在设置页展示尚不可用的插件开关或卸载按钮；
- 不实现账号、支付、订阅、分成、退款、许可证服务器或 DRM；
- 不承诺 Windows ARM64、macOS 或 Linux。

## 4. 设计原则

1. **默认拒绝**：未知来源、未知消息、未知 capability、未声明参数和失效身份均拒绝。
2. **最小权限**：正式插件权限按插件、发布者、major version 和具体资源范围绑定；开发包使用独立本地开发身份，不能继承正式插件身份或授权。
3. **边界可执行**：高风险权限必须由 OS 或宿主代理实际强制，不能只写在 manifest 中。
4. **来源不等于权限**：审核通过只说明来源与审核状态，不扩大运行时权限。
5. **进程不等于沙箱**：独立进程提供崩溃和卸载边界，只有验证过的 OS 限制组合才可形成隔离承诺。
6. **用户控制数据**：市场下架或撤销不静默删除用户数据。
7. **故障隔离**：插件崩溃、卡死、超限或拒绝退出不带崩宿主。
8. **可撤销**：授权、订阅、长期请求、进程和设备句柄必须能被宿主有界撤销。
9. **可回滚**：更新不覆盖当前版本，失败不得破坏最近可用版本。
10. **诚实表述**：产品只声明已经验证并可持续执行的隔离能力。

## 5. 受保护资产

### 5.1 宿主完整性

- LuckyIsland 主进程、安装目录和更新机制；
- Tauri command、managed state 和窗口政策；
- SQLite 主数据库和 settings KV；
- 市场信任根、撤销缓存、授权记录和激活版本指针。

### 5.2 用户隐私与凭据

- AI API Key、通知 Token 和其他密钥；
- 对话历史、语音内容和转写正文；
- 剪贴板、用户文件路径与内容；
- 插件私有配置、业务数据和日志。

### 5.3 设备资源

- 麦克风、网络、CPU、内存和磁盘；
- 快捷键、托盘、通知、窗口展示意图和子进程；
- 原生模型、运行时 DLL、CLI Provider 和文件句柄。

### 5.4 生态完整性

- 插件 ID 与发布者身份的绑定；
- 插件版本、内容哈希、签名、SBOM 和许可证；
- 市场索引、审核结果、安全公告、下架和撤销记录。

### 5.5 用户控制权

- 安装、授权、拒绝、撤销授权和卸载选择；
- 保留数据卸载与彻底卸载的明确区别；
- 对高风险模式、能力活跃状态和撤销状态的可见性。

## 6. 威胁主体与信任假设

### 6.1 威胁主体

本模型覆盖：

- 主动窃取数据、越权、持久化或逃逸的恶意插件作者；
- 账号或签名密钥被盗的合法发布者；
- 被篡改的插件包、下载镜像、市场索引或撤销缓存；
- 存在崩溃、死循环、资源泄漏或消息洪泛缺陷的良性插件；
- 尝试读取相邻插件数据或伪造身份的其他插件；
- 尝试篡改插件目录、IPC、缓存或激活版本的本机低权限进程；
- 误开侧载、忽略权限变化或安装同名仿冒包的用户；
- 明确接受来源风险的本地开发者模式使用者。

### 6.2 有限信任主体

以下主体只能获得有限信任：

- 市场服务可以签署审核结论，但审核不能代替运行时权限；
- 发布者签名可以证明内容来源，不能证明代码无恶意；
- 插件 manifest 是待验证声明，不能直接授权；
- 用户确认表达风险接受，但不能关闭宿主不可绕过的安全底线；
- 插件进程可以返回业务结果，不能自行声明权限、身份或安全状态。

### 6.3 攻击者能力边界

首版安全目标覆盖恶意插件本身、受限插件进程及低于 LuckyIsland 主进程完整性级别且没有同用户任意代码执行能力的本机进程。后续隔离验证必须显式记录进程 SID、完整性级别、AppContainer 身份、会话和可访问对象，不得笼统写成“本机低权限”。

首版不承诺防御：

- 已取得管理员、SYSTEM 或内核权限的攻击者；
- 与 LuckyIsland 同一用户、同一完整性级别并可执行任意本机代码的恶意进程；该进程可能尝试打开或注入宿主进程、窃取同用户可读 IPC 凭据，或修改用户可写状态；
- 用户主动把同一敏感信息分别授权给多个插件的行为。

因此，插件包、激活指针、撤销反回滚状态和授权记录仍必须采用签名、哈希、原子写和最小 ACL 防御偶然损坏与较弱攻击者，但不得宣称这些措施能抵御同用户同完整性级别任意代码执行。若未来要覆盖该攻击者，必须另行引入更高权限 broker/service 或等价、可验证的安全边界，并重新进行威胁建模。该限制必须在产品安全说明中如实披露。

## 7. 信任边界

### 7.1 LuckyIsland 主进程

主进程保存宿主策略、信任根、用户授权和生命周期状态，负责最终裁决窗口、资源和插件状态。

不可变底线：

- 第三方 DLL 或二进制不得动态加载进主进程；
- 插件不得调用任意 Tauri command；
- 插件不得直接读取宿主数据库或 managed state；
- 插件提交的窗口展示请求只是一种受限意图，最终由宿主 Window Policy 裁决。

### 7.2 Plugin Bridge

Plugin Bridge 是插件访问宿主能力的唯一入口，负责：

- 版本化握手、请求和事件 schema；
- 插件进程身份与当前启动实例绑定；
- manifest 声明、用户授权和实时策略检查；
- 参数、消息大小、并发、频率、deadline 和配额；
- 结构化错误、取消、撤销和脱敏审计；
- 插件停止后使旧 request、subscription 和临时身份失效。

Bridge 不接受自由字符串调用宿主任意方法，也不把内部 Tauri API 透传给插件。

### 7.3 插件执行域

插件执行域包括：

- WASM/WASI 实例；
- 每插件独立的受限原生 Host；
- 隔离 Web UI。

不同载体共享同一 capability 名称和用户授权语义，但由各自运行时落实限制。

### 7.4 外部资源

市场、网络、用户文件、麦克风、剪贴板和外部进程均位于边界外。插件只能通过对应 capability 和受控句柄访问，不能把获得一项能力解释为访问同类全部资源。

## 8. 三层策略裁决

### 8.1 安装裁决

```text
允许安装
  = 来源模式允许
  ∧ 包结构安全
  ∧ 所需签名有效
  ∧ 插件身份有效
  ∧ 平台与宿主兼容
  ∧ 用户确认权限和风险
```

### 8.2 启动裁决

```text
允许启动
  = 已安装且版本被激活
  ∧ 当前来源模式仍允许
  ∧ 签名、撤销和兼容状态有效
  ∧ 对应执行载体安全环境建立成功
  ∧ 当前授权满足最低启动需求
```

### 8.3 调用裁决

```text
允许能力调用
  = capability 已在 manifest 声明
  ∧ 用户已授予对应范围
  ∧ 当前插件启动身份有效
  ∧ 参数、目标和资源范围合法
  ∧ 未被撤销、限流或超额
  ∧ deadline 尚未到期
```

任何一层失败均拒绝，并返回稳定的结构化原因。上层成功不能覆盖下层失败。

### 8.4 权限声明与授权语义

安装确认只表示用户接受包来源和安装行为，**不等于授予 manifest 中的全部 capability**。Manifest 可以把能力标为运行必需或可选，但宿主维护可作为启动必需项的白名单；麦克风、剪贴板读取、进程启动、用户文件和敏感 AI 上下文等高风险能力不得仅因插件声明而在安装或启动时自动授予。

- 用户可以安装后拒绝部分可选能力，插件以降级功能运行；
- 真正缺少宿主允许设为必需的能力时，启动返回稳定的 `required_capability_missing`，不得循环弹窗或强迫授权；
- 高风险能力仍在首次实际使用时再次确认；
- 权限变化、范围扩大、发布者变化或 major version 变化时按策略重新授权；
- 插件不得通过把所有能力标为“必需”绕过分项同意，市场审核和宿主策略均应拒绝不合理的必需声明。

## 9. 安装来源策略

### 9.1 市场审核插件

市场插件必须具备：

- 有效发布者签名；
- 有效市场审核签名；
- 完整内容哈希与不可变版本；
- 插件 ID、发布者、版本、平台与内容的绑定。

市场审核不能绕过用户授权、OS 隔离、运行时策略、资源限制或撤销。

### 9.2 发布者签名侧载插件

所有正式发行版均提供可见但默认关闭的“允许不安全插件”入口。开启后可安装有发布者签名、但没有市场审核签名的包。

规则：

- 开启入口时展示持续风险说明；
- 每个包逐包确认发布者、来源、权限和风险；
- 仍执行完整包结构、路径、哈希、兼容性和隔离检查；
- UI 持续标记“未经 LuckyIsland 市场审核”；
- 不获得市场徽章，不进入市场自动更新通道；
- 关闭入口后停止自动启动依赖该模式的插件，但不删除程序或数据。

### 9.3 未签名本地开发包

所有正式发行版均提供可见但默认关闭的开发者模式。开启后允许安装未签名本地包。

规则：

- 开发包使用独立的 `dev.local/*` 身份命名空间，不能声明、覆盖或冒充市场/签名侧载插件 ID；
- 首次导入时由宿主生成项目身份，或使用本地开发身份密钥；后续授权、数据和更新关系绑定该开发身份，而非仅绑定可编辑的 manifest ID；
- 开发身份或本地开发密钥变化时，不继承原授权、密钥、数据或更新关系；如需迁移，必须走宿主明确的数据导入流程；
- 本地阻止列表至少可绑定开发身份、插件 ID 和内容哈希；身份冲突时拒绝安装，不覆盖既有身份；
- 开启时要求输入明确确认短语；
- 每个包仍需逐包确认；
- 应用升级或连续 30 天未使用后自动关闭；
- 只有成功执行明确开发操作才刷新最后使用时间，普通后台启动不刷新；
- 自动复锁后不再启动依赖开发者模式的插件，但不删除程序或数据；
- 开发者模式不能关闭进程隔离、capability、资源配额、本地阻止列表或高危撤销；
- 后续 SDK 应提供受限 Host 调试通道，而不是无隔离调试入口。

## 10. 执行载体策略

### 10.1 WASM/WASI

普通数据处理和自动化插件优先使用 WASM/WASI。默认不给予宿主网络、文件、进程、设备或 UI 权限；预开放的基础运行时能力也必须明确列入版本化契约。资源上限由运行时和宿主共同执行。

### 10.2 受限原生 Host

麦克风、原生模型、CLI 等确需原生能力的插件使用每插件独立原生 Host。原生 Host 必须在 Windows 受限运行环境中启动。隔离初始化失败时拒绝运行，不得降级为普通当前用户权限。

只有威胁模型和性能数据证明等价安全后，未来才可讨论按信任域共享 Host；首版不允许不相关第三方插件共享进程。

### 10.3 隔离 Web UI

插件 UI 不得注入宿主 React 树。隔离 UI：

- 不拥有全局 Tauri `invoke`；
- 不访问宿主 DOM 或全局 CSS；
- 不自由导航、打开外链或加载未声明来源；
- 只通过版本化消息桥访问能力；
- 只接收受控主题 Token、字号、减少动态效果和布局约束；
- 页面可见性、顺序、窗口尺寸、点击穿透和焦点仍由宿主裁决；
- 所有插件 UI 始终位于不可由插件绘制、覆盖或隐藏的宿主可信边框内，边框展示插件名称、身份和安装来源；
- 插件不得仿冒宿主权限、登录、签名、撤销或安全确认界面，不得在宿主安全确认期间获得置顶、全屏、无边框或焦点控制；
- 凭据输入、全屏/无边框展示和其他可弱化可信边框的能力不属于普通 `island.page`，必须另行设计和提高风险等级。

`island.page` 和 `settings.page` 只有在可信边框、布局限制、导航隔离和宿主安全 UI 不可覆盖全部成立时，才按低风险候选处理；否则提高风险等级。

## 11. Capability 风险模型

### 11.1 低风险

初始低风险候选：

- `storage.plugin`；
- `island.page`；
- `settings.page`；
- 受控主题 Token 和声明式 UI 元数据。

低风险能力可在安装时集中展示并授权，用户可随时撤销。

### 11.2 中风险

初始中风险候选：

- `network.domain`；
- `notification.publish`；
- `hotkey.action`；
- `tray.action`；
- `ai.provider`。

中风险能力必须限制具体域名、端口、参数、频率、内容大小、动作数量或贡献范围。`network.domain` 的连接和每次重定向都由宿主代理重新解析与裁决最终地址；默认拒绝 loopback、link-local、私有网段、保留地址、未经授权的 IP 字面量和本地代理绕过。CNAME 链、DNS 变化、TLS SNI/证书主机名与目标授权必须一致。本地网络访问是独立高风险范围，不能由普通域名授权隐含获得。

### 11.3 高风险

初始高风险候选：

- `microphone.capture`；
- `clipboard.read`；
- `process.spawn`；
- `storage.user_selected`；
- 涉及敏感上下文的 `ai.conversation`。

高风险能力除安装时展示外，首次实际使用时再次确认。活跃期间由宿主提供不可由插件隐藏的状态指示。撤销后立即拒绝新请求，并取消或终止可撤销的长期会话。

### 11.4 分级变更规则

后续威胁分析可以提高 capability 风险等级。不得在不重新授权的情况下静默降低等级、扩大范围或将原有精确权限合并为泛化权限。

## 12. Windows 原生隔离最低承诺

### 12.1 强制规则

第三方原生插件只在宿主成功建立并验证受限运行环境后启动。系统版本、文件系统、企业策略或其他环境条件导致隔离不可用时，返回结构化的不支持或隔离失败状态，不显示“仍然运行”按钮。

产品使用“受限运行环境”或“OS 隔离”表述。只有后续原型证明安全属性后，才可对相应实现使用“沙箱”一词。

### 12.2 必须验证的机制无关安全属性

“受限运行环境建立成功”不是某个 Windows API 调用返回成功，而是以下属性在对应 Windows 10/11 x64 支持版本的负向测试中全部成立：

- 默认不能联网；只有经宿主代理明确授权的目标可连接；
- 不能读取或修改宿主目录、宿主数据库、其他插件目录及普通用户文件；
- 只能访问本插件只读程序目录、私有数据目录和宿主显式传入的受控句柄；
- 不能打开、注入、调试或控制 LuckyIsland 主进程及其他插件进程；
- 不能访问未声明的注册表位置或敏感命名对象；
- 子进程不能脱离宿主控制的进程树，不能使用 breakaway 逃逸；
- 不继承白名单之外的文件、进程、令牌、网络或 IPC 句柄；
- IPC 只能连接当前启动实例，并验证双向身份、会话和新鲜度；
- CPU、内存、进程数、消息和请求频率可以被观察、限制并有界终止。

任一属性无法在某个受支持 Windows 环境中证明时，该环境不支持第三方原生插件。不能用提示框把缺失属性转化为用户可接受风险。

### 12.3 后续必须评估的机制组合

后续原生安全专项必须用上述属性评估而不是按 API 名称打勾：

- AppContainer 或受限 Token；
- Job Object 对进程树、子进程和资源的约束；
- 插件程序目录只读、插件数据目录独占的 ACL；
- 可实际执行域名限制的受控网络代理或系统网络策略；
- 用户选择文件或目录的受控句柄传递；
- `process.spawn` 的可执行目标、参数模板和工作目录约束；
- 每次启动生成的短期 IPC 身份和对端认证；
- CPU、内存、进程数、消息和请求频率限制。

不得把环境变量、工作目录、manifest 声明、进程名称或独立进程本身作为充分安全边界。

### 12.4 平台规则

- 首版只支持 Windows 10/11 x64；
- 安全基线取支持范围内可实际验证的共同能力；
- 宿主升级、插件 major 版本变化或相关系统策略变化后重新验证安全能力；
- Windows ARM64、macOS 和 Linux 不属于首版兼容承诺。

## 13. 插件生命周期

### 13.1 状态模型

后续实现使用显式状态机，而非单一 `enabled` 布尔值：

```text
Discovered → Verifying → Installed → Starting → Running
                                      ↓           ↓
                                   Disabled ← Stopping
                                      ↓
                                  Uninstalled

任意适用状态 → Quarantined → Revoked

更新临时链：Staging → HealthChecking → Activated / RolledBack / Failed
```

状态转换由宿主 Plugin Manager 裁决。数据库或等价原子状态记录当前激活版本；磁盘目录存在不代表版本已激活。

### 13.2 安装与更新

安装或更新流程：

1. 下载或读取临时包；
2. 验证来源要求的签名、哈希和身份；
3. 安全解析归档；
4. 校验 manifest、平台、宿主兼容性、权限、依赖和冲突；
5. 展示来源、权限变化和风险；
6. 解压到全新版本 staging 目录；
7. 在预激活隔离域中执行健康检查：默认无既有用户授权、无网络、无高风险能力、无实时插件数据目录，只能使用合成数据或只读/克隆快照；任何确需真实资源的检查必须单独说明并取得一次性授权；
8. 数据迁移只在 staging 副本或可回滚事务中执行，禁止预激活版本直接修改当前数据；
9. 健康检查和迁移通过后，原子切换代码与数据版本指针；
10. 保留至少一个最近可用版本；
11. 清理未激活 staging。

禁止覆盖当前运行版本、执行任意安装后脚本、只凭版本号判断身份、在发布者变化时静默接管插件 ID，或在健康检查失败后破坏当前版本。

不可逆数据迁移前必须备份并明确提示；需要不可逆迁移的版本不能宣称自动回滚，必须在激活前向用户说明回滚限制。新版本连续崩溃或健康检查失败时自动回滚或禁用。

### 13.3 停止协议

禁用、卸载和高危撤销共用停止协议：

1. 标记 `Stopping` 并立即拒绝新请求；
2. 取消后台任务、长期请求和订阅；
3. 移除 UI、快捷键、托盘动作和通知源；
4. 请求插件正常退出；
5. 有界等待后终止完整进程树；
6. 验证麦克风、文件、模型和子进程句柄已释放；
7. 按操作类型删除程序、缓存或数据；
8. 文件占用时登记宿主重启后的待清理任务；
9. 重启后复核残留并向用户报告。

插件不得提供任意卸载脚本。

### 13.4 数据生命周期

**保留数据卸载**删除程序、运行时、模型和可再生缓存，保留设置、密钥和业务数据。

**彻底卸载**进一步删除插件私有数据、用户授权和普通日志，但不删除用户主动导出到插件目录外的文件，也不删除宿主维护的撤销、反回滚最高 epoch、身份占用、内容阻止和安全处置记录。执行前展示预计删除项和大小，并进行明确确认。

卸载删除绝不跨越经验证的插件根目录。清理以已验证根目录句柄和文件身份执行，不跟随 reparse point、junction、符号链接或挂载点，只删除链接本身；检测硬链接、目录对象替换或边界不确定时停止删除并报告残留，不能为追求“彻底”而递归进入边界外路径。

数据分为四类：

1. 插件私有数据：按保留数据卸载或彻底卸载选择处理；
2. 用户授权：彻底卸载删除，重装必须重新授权；
3. 普通审计日志：可由用户清除，受容量与保留期约束；
4. 宿主不可删除安全状态：撤销、反回滚、身份占用和处置记录，不随插件卸载或普通日志清除而删除，按宿主安全政策保留。

高危撤销后的数据导出只能由宿主导出原始或已知结构化数据，不得为了调用插件自定义导出逻辑而重新启动已撤销代码；专有格式需等待后续受信任迁移工具。

## 14. 下架、撤销与离线策略

### 14.1 普通下架

普通下架：

- 阻止新安装和更新；
- 已安装版本继续运行；
- 显示下架原因和安全公告；
- 不自动删除程序或数据。

### 14.2 高危撤销

收到有效签名高危撤销后：

- 立即拒绝新调用；
- 移除扩展点并终止运行中的插件进程树；
- 阻止对应发布者密钥、插件 ID、版本或内容哈希再次启动；
- 提供数据导出、保留数据卸载和彻底卸载；
- 不自动删除用户数据。

撤销粒度必须支持发布者密钥、插件、版本和精确内容哈希。

### 14.3 离线裁决

市场不可用时使用本地签名缓存。缓存具有生成时间、有效期、签名链和单调递增的 epoch/sequence。宿主在独立于普通日志和插件数据的不可删除安全状态中保存最高已接受 epoch，拒绝任何更低版本；缓存还必须遵守最大寿命，不能因系统时钟回拨无限延长：

- 缓存签名有效、epoch 不低于本地最高值、未超过最大寿命且时钟可信时，按缓存裁决；
- 系统时间明显回拨、反回滚状态损坏或无法证明缓存新鲜度时按缓存失效处理，高风险和原生插件 fail closed；
- 缓存过期后，低风险 WASM 插件可继续运行并持续显示安全状态过期；
- 原生插件或拥有高风险能力的插件暂停启动；
- 已运行的高风险插件进入有界停止流程；
- 普通设置不能绕过过期暂停；
- 未签名本地开发包不依赖市场撤销缓存，但仍受本地阻止列表、隔离和权限策略；
- 签名侧载插件若发布者或内容哈希进入有效撤销列表，同样阻止运行。

短时市场故障不得导致所有低风险已安装插件失效，也不得忽略已有有效高危撤销。

### 15.1 身份层级

市场展示三个互不混淆的身份状态：

## 16. 隐私、遥测与审计

### 16.1 遥测

插件与市场健康遥测默认关闭，由用户主动加入。用户可查看收集类别、用途和保留周期，并可随时退出。退出后停止新的遥测上传，不影响本地安全审计。

市场索引、有效撤销列表和安全公告属于必要安全通信，不归类为遥测，但仍遵循最小请求原则。

允许用户选择上传的非敏感指标仅限：

- 安装、更新、回滚和卸载结果类别；
- 启动耗时区间、崩溃摘要和资源超限；
- 宿主/协议兼容失败类别；
- 插件与宿主版本、Windows 大版本和体系架构。

禁止默认上传对话、Prompt、模型回复、音频、转写、密钥、Token、Cookie、请求正文、剪贴板、文件名、完整路径、文件内容、表单输入、插件私有数据或设备指纹。

### 16.2 本地安全审计

无论遥测选择如何，宿主保留本地、脱敏、可轮转的安全审计：

- 插件身份、版本和来源；
- capability、时间、结果类别和拒绝原因代码；
- 非敏感参数摘要，例如目标域名而非完整 URL 查询；
- 启停、崩溃、超限和撤销；
- 权限授予、拒绝、撤销和高风险模式变化。

日志不记录密钥、音频、对话、剪贴板或文件正文；具有容量和保留时间上限；插件不能修改其他插件或宿主日志。用户可查看、导出和清除普通日志，但清除动作不删除维持高危撤销和策略完整性所需的最小安全状态。

### 16.3 用户可见安全状态

插件详情最终需要展示：

- 安装来源和签名状态；
- 发布者验证等级；
- 执行载体和隔离状态；
- 已声明、已授予和已拒绝的 capability；
- 高风险能力最近使用时间和当前活跃状态；
- 网络域名、用户选择文件范围和进程模板；
- 撤销缓存状态和最后更新时间；
- 资源使用、最近异常、数据目录大小和卸载影响。

高风险能力活跃指示由宿主控制，插件不能隐藏或伪造。

## 17. 性能预算原则

阶段 0 不在缺少原型数据时锁定精确 MB、CPU 百分比或毫秒上限。阶段 1 及后续原型必须测量并校准：

- 无插件时 Plugin Manager 的启动、空闲内存和 CPU 增量；
- 单个空闲 WASM 插件；
- 单个空闲原生 Host；
- 单个隔离 Web UI；
- 冷启动、恢复和停止耗时；
- 消息吞吐和大 payload；
- 崩溃重启与循环保护；
- 多插件并存总预算。

每项最终预算包含正常目标、警告阈值、强制限制、观察窗口、超限动作和 Windows 10/11 x64 测试环境。

超限处置递进为：

```text
记录 → 限流 → 暂停后台任务 → 停止插件 → 自动禁用
```

宿主不得因一个插件超限失去响应。原生进程树和 WASM 实例都必须可被宿主有界终止。性能分析不得成为上传敏感正文的理由，隐私策略也不得取消本地资源计量。

## 18. 安全需求与威胁映射

| 威胁 | 必需控制 | 失败结果 |
|---|---|---|
| 第三方代码破坏宿主 | 禁止主进程加载；进程外执行 | 拒绝加载或启动 |
| 伪造插件身份 | 内容签名、短期启动身份、IPC 对端认证 | 拒绝握手 |
| 越权调用宿主 | Bridge 白名单、schema、授权和参数校验 | 结构化拒绝并审计 |
| 读取其他插件或宿主数据 | 独立数据目录、ACL、无数据库直连 | 拒绝访问并隔离 |
| 任意联网 | `network.domain`、重定向复核、可执行网络限制 | 拒绝目标请求 |
| 任意进程启动 | 固定目标、参数模板、受控目录、进程树约束 | 拒绝启动 |
| 资源耗尽 | 配额、限流、Job Object/运行时限制、有界终止 | 限流、停止或禁用 |
| 恶意更新 | 双签名、不可变哈希、staging、健康检查和回滚 | 保留当前版本 |
| 发布者密钥泄露 | 密钥撤销、精确版本/哈希阻止、签名公告 | 高危停止并阻止启动 |
| 市场短时不可用 | 有效签名缓存和分级离线策略 | 低风险继续，高风险按有效期暂停 |
| 卸载残留 | 统一停止协议、进程树终止、重启后清理 | 报告残留，不虚报完成 |
| 侧载绕过安全 | 侧载只改变来源信任，隔离与 capability 不可关闭 | 拒绝不满足底线的包 |

## 19. 场景验证矩阵

阶段 0 通过规则推演验收，不声称产生运行时测试结果。以下场景必须从本规格得到唯一裁决。

### 19.1 安装来源

- 市场签名有效但发布者签名失效：拒绝安装；
- 发布者签名有效但市场签名缺失：仅在“允许不安全插件”开启且逐包确认后按侧载安装；
- 未签名包且开发者模式关闭：拒绝安装；
- 应用升级后开发者模式复锁：依赖该模式的插件不再启动，数据不删除；
- 同一插件 ID 由不同发布者签名：拒绝，除非完成显式所有权迁移；
- 同版本号对应不同内容哈希：拒绝并记录供应链异常；
- 路径遍历、绝对路径、符号链接逃逸、大小写碰撞或压缩炸弹：在解压激活前拒绝。

### 19.2 启动和隔离

- Windows 10/11 无法建立原生受限环境：拒绝启动，不提供降级按钮；
- 原生插件直接访问宿主数据库或其他插件目录：由 OS/ACL 阻止，并记录违规；
- 其他本机进程伪造 IPC 身份：握手失败；
- 原生插件派生子进程并拒绝退出：有界等待后终止完整进程树；
- WASM 请求未开放宿主能力：Bridge 拒绝；
- Web UI 调用全局 Tauri `invoke`、注入宿主 CSS 或自由导航：隔离容器和 Bridge 拒绝；
- 宿主升级导致协议或隔离不兼容：插件进入不兼容状态，不尝试不安全启动。

### 19.3 权限调用

- 未声明 capability：拒绝；
- 已声明但用户未授权：拒绝；
- 长期请求期间撤销权限：拒绝新调用并取消可撤销会话；
- 网络重定向到未授权域名：重定向目标重新裁决并拒绝；
- `process.spawn` 替换可执行文件或注入 shell 参数：身份/模板校验失败并拒绝；
- 用户选择句柄过期或超出范围：拒绝并要求重新选择；
- 消息洪泛、超大 payload、无界订阅或超时：限流、取消、停止或自动禁用。

### 19.4 更新和撤销

- 新版本健康检查失败：不切换激活版本并清理 staging；
- 发布者密钥正常轮换：验证轮换链并按政策重新评估授权；
- 发布者密钥泄露：应用有效高危撤销并阻止受影响版本；
- 普通下架：阻止新安装和更新，已安装版本继续；
- 高危插件运行中收到撤销：立即拒绝新调用并有界停止；
- 市场不可达但撤销缓存有效：按缓存裁决；
- 撤销缓存过期：低风险 WASM 警告后继续，原生或高风险插件停止并暂停启动；
- 侧载包的发布者或哈希进入有效撤销：阻止运行。

### 19.5 数据和卸载

- 保留数据卸载后重装同一可信插件：可恢复保留数据；
- 彻底卸载遇到占用文件：登记重启清理并报告未完成状态；
- 插件拒绝退出或遗留子进程：终止进程树后再清理；
- 不可逆数据迁移：先备份并明确确认；
- 用户导出文件位于插件目录外：卸载不删除；
- 卸载后快捷键、托盘、UI、订阅或设备句柄残留：卸载判定失败并报告残留。

### 19.6 安全复核增补场景

- 同用户、同完整性级别任意代码执行者尝试注入宿主：明确属于首版模型外，不得以插件隔离测试声称已防御；
- 候选原生隔离只限制 Token 和进程树、但仍可读普通用户文件或任意联网：隔离属性不完整，判定该环境不支持原生插件；
- 未签名开发包声明已有市场插件 ID：因命名空间冲突拒绝，不继承数据、密钥或授权；
- 开发身份密钥变化：视为新身份，旧授权和数据不自动继承；
- 恢复签名有效但 epoch 更旧的撤销缓存：拒绝；系统时钟回拨或最高 epoch 状态损坏时高风险插件暂停；
- 新版本健康检查尝试读取实时数据、联网或使用旧版高风险授权：预激活隔离域拒绝；
- 插件数据目录包含 junction、reparse point、硬链接或删除时目录被替换：不跟随，停止不确定删除并报告残留；
- 已授权域名解析到 loopback、私网、link-local，或经 CNAME/重定向改变目标：宿主代理重新裁决并默认拒绝；
- 用户安装时接受来源但拒绝可选权限：允许安装并以降级能力运行；插件将高风险能力标为启动必需：宿主策略拒绝不合理声明；
- 插件 UI 仿冒宿主授权框、覆盖安全提示或诱导输入凭据：可信边框保持可见，宿主阻止覆盖/焦点控制并记录违规；
- 彻底卸载后重装已撤销插件：宿主不可删除安全状态仍阻止启动；
- 高危撤销后请求专有数据导出：不重新启动已撤销代码，只提供宿主可安全执行的原始/结构化导出。

## 20. 阶段 0 完成门禁


阶段 0 完成需要：

- 规格不存在未决占位、相互矛盾的安全降级或含糊授权；
- 验证矩阵各类场景均有唯一预期；
- 主 Agent 完成占位、矛盾、范围和歧义自检；
- 最多一个独立只读 Agent 完成安全复核；
- 高严重度复核问题已修复，或由用户明确拒绝并记录理由；
- 用户审阅书面规格并最终批准。

该门禁不等于插件功能完成，也不授权阶段 1 编码。

### 20.1 独立安全复核记录

2026-07-14 使用唯一一个独立只读安全审查 Agent 对本规格进行两轮复核，未检查或修改无关工作区文件。

初审提出 LI-SEC-001～LI-SEC-010：同用户攻击者边界、原生隔离可测试属性、开发包身份、撤销反回滚、预激活健康检查、安全删除、DNS/私网绕过、安装与授权语义、可信 UI、不可删除安全状态。规格逐项修订后，复审确认 10/10 已解决，未发现新增高或中严重度问题，最终门禁为 **Pass**。

该 Pass 只证明阶段 0 安全规格具备进入用户最终审批的条件，不证明插件运行时已经实现，也不授权阶段 1 编码。

## 21. 阶段 1 边界

阶段 0 最终批准后，阶段 1 必须另行 brainstorming、设计和编写实施计划。

阶段 1 只允许设计：

- 版本化 manifest/schema；
- 版本化 IPC、握手、request ID、deadline 和结构化错误；
- 一个无网络、无原生能力的 LuckyIsland 官方示例插件；
- 插件进程启动、停止、基本健康检查和崩溃隔离；
- 未知调用默认拒绝。

阶段 1 不允许：

- 开放第三方原生插件；
- 实现公开市场、侧载或完整权限系统；
- 实现 AppContainer/受限 Token 等原生隔离承诺；
- 迁移 AI、语音、热键、通知或设置页；
- 把本路线图或本规格直接转换为跨阶段代码任务。

## 22. 后续必须验证的技术问题

阶段 0 有意不锁定下列实现选择，后续候选必须满足本规格：

- WASI 运行时及其资源限制能力；
- RPC 编码、传输和大数据流机制；
- AppContainer 与受限 Token 的具体组合；
- Job Object、ACL 和句柄传递实现；
- 域名级网络代理或系统网络策略；
- 隔离 Web UI 的 WebView/iframe 边界和 CSP；
- 签名算法库、信任根和密钥存储；
- 精确资源预算；
- 市场服务托管、可用性和运营成本。

技术原型若无法满足强制规则，必须回到产品设计重新裁决，不能通过实现细节静默削弱本规格。

## 23. 已批准产品决策汇总

| 决策 | 结论 |
|---|---|
| 专项拆分 | 阶段 0 单独立项；阶段 1 另写规格与计划 |
| 第三方原生插件 | 审核通过后允许，但必须强制建立受限运行环境 |
| 不安全加载 | 两级入口：签名侧载 + 未签名开发者模式 |
| 不可关闭底线 | 第三方二进制不进主进程；包检查、隔离、capability 和资源约束持续有效 |
| 平台 | Windows 10/11 x64 |
| 执行载体 | WASM/WASI + 受限原生 Host + 隔离 Web UI |
| 防误用 | 逐包确认；开发者模式确认短语；升级或 30 天未使用自动复锁 |
| 撤销 | 普通下架与高危撤销分级；高危立即停止，不删数据 |
| 离线 | 缓存过期后低风险 WASM 警告继续；原生/高风险暂停 |
| 发布者身份 | 基础验证、身份验证、官方三层 |
| 正式版侧载 | 所有正式版均提供可见但默认关闭的入口 |
| 首版付费 | 仅免费插件 |
| 遥测 | 默认关闭，用户主动加入 |
| 性能预算 | 阶段 0 定原则，后续原型校准精确数值 |
| 阶段 0 评审 | 用户批准 + 最多一个独立只读安全复核 |

## 24. 完成标准

本设计只有在以下条件满足时，才可标记阶段 0 完成：

- 信任边界、资产、攻击者和安全目标明确；
- 来源、载体和能力三维策略不存在万能豁免；
- Windows 原生隔离失败时的拒绝行为明确；
- 两级侧载、撤销、离线和数据规则可一致执行；
- 市场治理、隐私、审计和性能规则可供后续实现验证；
- 验证矩阵覆盖安装、启动、调用、更新、撤销和卸载；
- 阶段 1 的范围和禁止项明确；
- 独立复核与用户书面审阅完成。

阶段 0 完成只代表安全与产品规则获批，不代表任何插件功能已经实现。
