## 接口与类型
```ts
interface NotifyInput {
  title: string;
  body?: string;
  source: "claude" | "codex" | "custom";
  level: "info" | "success" | "warn" | "error";
  action?: { type: "open_terminal"; cwd: string };
}
```
Rust：本地 HTTP `POST 127.0.0.1:9753/notify`（token 鉴权）；命名管道接收 CLI
事件：`notify://incoming`、`notify://history`

## 实现要点
- 主进程 `axum` 起 `127.0.0.1:9753`，token 从 env `LUCKY_TOKEN`
- `lucky-notify` CLI：独立 binary，连命名管道推消息；主进程离线时存为待发送
- 通知到达：岛展开 + 跳通知页 + 卡片滑入 + 系统通知（可关）+ 可选声音
- 通知历史存 SQLite `notifications` 表
- Claude Code `Stop` hook 模板 + Codex completion hook 模板（docs 提供）
- 专注模式下通知仍能浮现

## 测试要点
- `lucky-notify --title x --source claude` 触发岛通知
- `curl POST /notify` 带 token 触发
- 通知历史可回看
- 错误 token 拒绝
