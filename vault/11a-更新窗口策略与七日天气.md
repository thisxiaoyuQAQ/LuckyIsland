## 本轮范围

### 实施（全部为增量）

1. 可开关悬浮胶囊模式：默认关闭；开启后收起态为 240×80 `capsule`，右侧悬停先变为 720×80 `compact`，显式单击进入 720×400 `expanded`。
2. 默认关闭的悬停自动展开：普通条状仅在开关开启时由右侧悬停展开；浮球与该开关同时开启时采用胶囊→条状→完整面板的两段交互；移入约 180ms，移出约 300ms，穿透时暂停。
3. 灵动岛经典/新视觉样式切换：默认及非法值回退新样式；新样式移除紧凑态旧 leading 页点容器、放大紧凑时钟，并在内容右侧显示滚动指示。
4. 灵动岛左侧滚轮复用现有启用页面和顺序循环切换内容；左侧悬停绝不触发展开，右侧作为独立悬停/操作命中区。

## 已冻结产品决策

基线冻结决策（WindowPolicy 统一意图、用户 Hidden 最高优先级、穿透默认关、全屏检测语义、通知 priority 语义、stable 更新策略、天气供应商与缓存规则）已实现并随基线验收，继续有效，本文不重复。以下为增量冻结决策：

- 悬浮胶囊默认关闭；开启后 compact 意图的休眠投影是 240×80 `Capsule`，不是保留 720px 原生透明命中区的 CSS 缩放。
- 悬浮胶囊的右侧悬停固定执行 `Capsule → Compact`；显式单击可立即进入 `Expanded`。
- 悬停自动展开默认关闭；浮球关闭时仅控制右侧 `Compact → Expanded`；与浮球同时开启时，持续停留右侧按两段交互继续 `Compact → Expanded`。
- 左侧区域只复用现有页面滚轮切换，不提交窗口状态；左侧悬停绝不展开。右侧命中区稳定独立，设置/主题等按钮阻止冒泡。
- 主动展开后鼠标移出不收起；快速掠过、相反事件、关闭开关、穿透和卸载必须取消旧 timer/generation。
- 灵动岛视觉样式为 `legacy | new`；默认、缺键和非法值均为 `new`。经典样式保持现状；新样式不渲染旧 `flex items-center gap-1.5` leading 页点容器，放大 compact 时钟并把动画滚动指示放到内容右侧。
- 所有窗口宽度变化以相同顶部中心锚点重新定位；hover 引起的窗口变化不得抢焦点。

## 接口与类型

基线命令与事件（`set_island_state`、`window_policy_get`、`window_click_through_set`、`window_hover_expand_set`、`window_hide_in_fullscreen_set`、`window_policy_reload_settings`、`window://state-changed`、`window://policy-changed`）保持不变。以下为增量 diff；跨 Rust/TypeScript 的命令、事件和 DTO 必须保持一致。

## 分阶段任务

### 11a.1 策略层扩展

- 在既有纯归约器与 `WindowPolicy` 上扩展 `Capsule`、`floating_ball`、`hover_stage`；平台调用不得跨锁执行。
- `WindowPolicy` 统一管理 Capsule 240×80、Compact 720×80、Expanded 720×400；尺寸变化保持顶部中心锚点，监视器 clamp/恢复使用实际几何。
- 浮球与 hover stage 使用专用策略命令；平台失败不落盘错误状态、不广播未生效状态，UI 回滚到实际快照。
- 配置导入后重载 Rust policy；新增 `window:*` 设置保持 portable。
- 先写归约单元测试，再接平台效果。

### 11a.2 悬浮胶囊与左右分区交互

- 前端外壳拆为左侧内容/wheel 区与右侧 hover/action 区；左侧复用 `App.tsx::setPage`，绝不提交 hover 状态。
- 前端用 generation/可取消 timer 提交右侧 hover stage，禁止直接改窗口状态；主动 expanded 优先于 pointer leave，hover resize 不 focus。
- 移入约 180ms、移出约 300ms；穿透时暂停；快速掠过、相反事件、关闭开关和卸载必须取消旧 timer/generation。
- Capsule 为真实 240×80 原生窗口命中区，透明区域不拦截后方点击。

### 11a.3 双视觉样式与紧凑内容导航

- `src/lib/settings.ts` 增加 `window:floating_ball`、`window:visual_style` 的键、默认值和严格 parser；`VisualStyle` 仅接受 `legacy | new`，其余回退 `new`。
- `src/lib/window-policy.ts` 集中策略快照 IPC、两段 hover controller、wheel 方向/节流与 legacy/new style recipe，并先补 Node Vitest 纯逻辑测试。
- 经典样式保持当前紧凑页点和 `text-sm font-medium tabular-nums` 行为；新样式移除 leading 页点容器，仅放大 compact 时钟，并在当前紧凑内容右侧渲染绑定页面 index/count 的动画滚动指示。
- 单页时不显示误导性的滚动提示；展开页和终端内部滚动不被外壳 wheel 劫持。
- General 设置浮球/悬停开关，Appearance 设置经典/新样式；跨窗口变化和配置导入后 UI 必须与实际策略/设置同步。
