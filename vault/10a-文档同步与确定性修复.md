**失败场景**：应用不重启、时间页不卸载并跨过本地午夜后，木鱼“今日功德”和心情日期/连续天数仍使用旧日。

| 步骤 | 结论 | 落盘时间 |
|---|---|---|
| 1. 复现 | ✅ 新增 23:59 → 00:00 测试；首次运行 Vitest 正常启动，但因 `../localDayStore` 不存在而失败（1 failed suite，0 tests），证明缺失共享日期边界能力。 | 2026-07-13 23:59 |
| 2. 定位 | ✅ 木鱼只在 mount 时 `rolloverMerit`；心情在 render 固定 `localDateKey(new Date())` 且 effect 只执行一次。两者均没有午夜调度或可见性补偿。 | 2026-07-14 |
| 3. 修复 | ✅ 新增可注入环境的模块级 `LocalDayStore` 与 `useSyncExternalStore` hook；单一午夜 timer、visible 补偿、最后订阅者清理。木鱼按新日期 rollover 并立即持久化，心情按新日期重载并丢弃旧日异步结果。未修改 FIX-10A-02 的卸载 flush 语义。 | 2026-07-14 |

**验收结论**：跨日自动切换、恢复可见补偿、清理/重新订阅与现有日期逻辑回归均通过；下一项为 FIX-10A-02。

### FIX-10A-02 木鱼卸载 flush

> 状态：✅ 已完成（2026-07-14）
> 来源：AUD-FIX-02

**失败场景**：快速敲击后 500ms 内切页/卸载，cleanup 取消防抖保存，尾部计数丢失。

**四步原则记录**：

| 步骤 | 结论 | 落盘时间 |
|---|---|---|
| 1. 复现/保护 | ✅ 纯 writer 测试覆盖 500ms 到期前调用 `flush()` 只写入 latest。原始 cleanup 缺陷没有保留可引用的历史 RED 命令输出，因此不补造；留存 RED 依次证明：定时器触发的 latest 写入失败后未被 `flush()` 重试（expected 2 writes, got 1）、旧失败值会在更新值成功后残留，以及并发 `flush()` 会重复重试相同 latest 失败值（actual 4 writes, expected 3）。loader 测试另覆盖初始设置读取 rejection，以及设置读取跨过午夜后必须使用读取完成时的本地日期。 | 2026-07-14 |
| 2. 定位 | ✅ 旧 cleanup 只执行 `clearTimeout`，尚未触发的 pending 最新功德值会被丢弃；writer 的失败状态和并发 `flush()` 还可能造成漏重试、旧值残留或重复重试。初始读取 rejection 若未收口会令组件无法完成加载；若在读取前固定日期，跨午夜读取会按旧日归档。 | 2026-07-14 |
| 3. 修复 | ✅ 使用组件实例私有的 `DebouncedWriter`：`schedule()` 合并 500ms 内的最新值，写入链和并发 `flush()` 严格串行，失败只保留当前最新值供后续重试，更新值成功后清除更旧失败状态；木鱼卸载和跨日调用 `flush()`。加载器在设置读取完成后调用 `currentLocalDay()`，运行时校验存量结构；读取 rejection/畸形存量保持禁用以防覆盖未知历史，加载前点击同时由按钮和处理函数保护；点击本身也先按实时日期 rollover。 | 2026-07-14 |
| 4. 验证 | ✅ writer 8/8；woodenFishState 6/6；writer + woodenFishState + localDayStore + date 聚焦测试 31/31；完整 Vitest 106/106；`pnpm exec tsc --noEmit`、三入口 Vite build 和全量 `git diff --check` 通过。构建仅有既有主 chunk >500 kB 警告。 | 2026-07-14 |

**实现边界**：

- 正常敲击仍是 500ms debounce，不改成每击 IPC。
- `DebouncedWriter` 由每个 `WoodenFishWidget` 实例私有持有，不是跨组件实例共享队列；不宣称新挂载会等待旧实例 writer 排空。
- 写入链与 `flush()` 调用串行；定时写失败可由后续 `flush()` 重试，并发 `flush()` 不重复重试相同失败值，更新值成功后不会继续重放更旧失败值。
- 初始设置加载完成前，按钮处于 disabled 状态，`knock()` 也会直接返回；读取 rejection 或合法 JSON 中字段畸形时显示安全零值但保持禁用，避免后续写入覆盖未知存量；只有读取成功且缺键或存量结构有效时解除保护。
- loader 在异步设置读取完成后才调用 `currentLocalDay()`；设置读取跨过午夜时按读取完成后的日期 rollover，不沿用读取开始前的旧日期。
- 卸载 cleanup 异步发起 `flush()` 并吞掉该调用错误，因为卸载期间无法再安全展示错误；不把它描述成跨实例强一致性保证。
- 本项有纯 writer、loader 和相关日期逻辑回归；未额外执行 GUI 快速切页真机观察，不把它写成已完成真机证据。

**验收结论**：卸载 cleanup 不再只取消 timer，而会把当前组件实例的 pending 最新值提交给串行写入链；定时失败、旧失败状态、并发 flush、加载失败和异步读取跨午夜均有回归保护，正常 debounce 保持。按用户确认的续接顺序，模块 11 Task 2 已完成，当前入口为 Phase 1 / Task 3；FIX-10A-03 保留排队。
