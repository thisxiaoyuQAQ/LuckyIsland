## Bug 摘要
- 实际表现：LuckyIsland 中使用 Codex CLI 询问“兰州现在多少度”时，完整问题已通过 stdin 送达，但请求在 120 秒后被终止。
- 期望表现：实时问题必须先取得真实联网结果，再由轻量 Codex 在 120 秒内回答；没有联网证据时明确失败，禁止按模型记忆猜测。
- 复现路径：UI 选择 Codex CLI → 发送天气问题 → Codex 继承 `xhigh`、全局 OMC/skills/MCP 后绕行 shell/browser，原生搜索不可用，最终触发后端 120 秒超时。
- 影响范围：Codex CLI 的天气、新闻、股价、汇率及“最新/当前”等实时问答；Claude CLI 与 Chat API 不直接经过本次 Codex 参数链路。

## 根因假设
- 假设：当前 custom Codex provider 不提供原生 Responses `web_search`，而现有调用仍把 Codex 当完整编码 Agent 启动；`xhigh`、全局 skills/MCP 和工具绕路共同放大耗时。
- 证据 1：对应 Codex JSONL 会话完整收到“兰州现在多少度”，21:12:12 启动，约 120 秒后仍未生成最终答案，期间依次尝试多个浏览器/Node/shell 路径。
- 证据 2：轻量真实探针使用 `--search --json --ephemeral`、`model_reasoning_effort="low"` 且禁用 codegraph/node_repl，13.339 秒明确返回 `WEB_SEARCH_UNAVAILABLE`，没有 `web_search` 事件。
- 证据 3：`--ignore-user-config` 的官方 provider 对照探针约 130 秒后认证失败，当前不能作为联网兜底。

## 选定方案
- 方案：A+——保留轻量 Codex，但将联网职责前置到 LuckyIsland 后端。实时问题先调用已有天气/搜索网络链路，成功后把带来源的检索证据注入 Codex prompt；检索失败则不启动 Codex并返回明确错误。
- Codex 轻量化：强制低 reasoning、临时会话、禁用现有 MCP，避免继承编码任务所需的重工具路径。
- 联网保证：以“后端网络调用成功并取得非空结果”为成功门槛，而不是仅检查 `--search` 参数；无证据不接受实时答案。
- 改动范围：仅 `src-tauri/src/ai/provider.rs` 及其测试；必要时同步本文件、BUG-04 和开发进度。
- 风险点：实时问题关键词需覆盖常见中文/英文表达；DuckDuckGo 结果质量依赖外部网络；非实时普通问答不应被强制搜索。
- 不在范围：不调整 KWS/VAD/ASR/TTS；不修改录音提示；不改前端；不提交本轮改动。

## 修复结果
- Codex 实时问题现在先在 LuckyIsland 后端分类：天气走 `uapis.cn` 结构化天气 API；新闻、行情、最新信息走 DuckDuckGo HTML 搜索；非实时问题不强制联网。
- 联网成功且取得非空证据后才启动 Codex；证据被标记为实时事实唯一依据，并明确禁止再次搜索或用模型记忆改写实时数值。联网失败会返回“未调用 Codex”，不再降级为猜测。
- 新鲜验证：Codex 定向测试 8/8、AI 模块 17/17，`cargo check`、`npx tsc --noEmit`、定向 `rustfmt --check`、`git diff --check` 均 exit 0；真实天气 API 返回兰州 19°C/阴，DuckDuckGo 返回 HTTP 200、10 条摘要。
- 2026-07-10 端到端复验曾被全局 Codex 配置中的 `[features].notify` 类型错误阻断；配置恢复后已于 2026-07-11 完成 LuckyIsland/Codex CLI 真机验收，实时天气问答能够返回联网结果且未再超时，BUG 关闭。
