## 接口与类型

### 新增 KV 约定（与 `settings.ts` KEYS 对齐）
| key | 取值 | 默认 | 说明 |
|---|---|---|---|
| `window:opacity` | "0.10"~"1.00" | "0.7" | 岛容器 CSS 背景透明度（alpha）；默认保持原 `bg-card/70` 视觉 |
| `window:offset_x` | 整数字符串，可为负 | "0" | 相对默认「顶部居中 + top gap」位置的横向物理像素偏移 |
| `window:offset_y` | 整数字符串，可为负 | "0" | 纵向物理像素偏移（负=上移，正=下移） |

> 透明度**不在 Rust 用**，纯前端读写 + CSS；偏移**必须 Rust 用**（`set_position` 物理像素）。

### 新增 Rust 命令（`settings_window.rs` / `monitor.rs`，不改 `settings.rs`）
```rust
// monitor.rs：实时应用偏移（不切屏），供设置面板滑块拖动时实时预览
window_offset_apply(app, db, offsetX: i32, offsetY: i32) -> Result<(), String>
//   写 window:offset_x/y + 立即按当前屏重算 set_position；带 clamp 防跑出屏

// settings_window.rs：配置导入导出
config_export(app, db, path: String) -> Result<(), String>
//   读可安全迁移的 settings 白名单 + stock_watchlist + weather_cities，写 JSON 到 path
config_import(app, db, path: String) -> Result<ImportSummary, String>
//   读 JSON → 校验 version → 事务内覆盖三表 → 逐条 emit settings://changed
//   → 触发 window_offset_apply 让偏移真正上屏
```

```rust
pub struct ImportSummary { settings: usize, watchlist: usize, cities: usize }
```

### 导出 JSON schema（version=1）
```json
{
  "version": 1,
  "exportedAt": "<前端传入的 ISO 字符串>",
  "app": "com.luckyisland.app",
  "settings": { "key": "value", ... },
  "stockWatchlist": [{ "symbol": "sh600519", "sort": 0 }, ...],
  "weatherCities": [{ "city": "...", "sort": 0, "addedAt": 123 }, ...]
}
```

### 事件
- 复用 `settings://changed { key, value }`：导入时对每个被覆盖的 setting key 逐条广播，各页已有监听即自动重算
- 偏移变更额外调 `window_offset_apply` 让窗口真的移动（纯数据写入不上屏）

## 实现要点

### 偏移（monitor.rs，最小侵入）
- `top_center_position(monitor, window_width, offset_x, offset_y)` 追加两个偏移参数：`x = base_x + offset_x`、`y = base_y + offset_y`（`base_y = monitor.position.y + TOP_GAP_PHYSICAL`）
- 新增 `read_offsets(db) -> (i32, i32)`：读 `window:offset_x/y`，无效回 0/0
- `move_island_to_monitor` / `recover_island_to_monitor` / `restore_island_monitor` 调用处传入 offsets
- `monitor_select` 切屏时也带上 offsets（切完仍偏移）
- `window_offset_apply`：不切屏，仅按当前 resolved monitor + offsets 重算 `set_position`，节流由前端做（200ms）；写 KV 后不广播窗口移动事件（窗口自己动了，前端状态不依赖它）
- **clamp**：偏移后的窗口左上/右下角至少要留在 resolved monitor 可视区内。按 `monitor.position/size` 与窗口物理宽高算上下限；超出 clamp 到边界。极端偏移（如 5000）不会把岛推到屏外消失
- 单测：`top_center_position` 带 offset 的几种组合（正/负/超界 clamp）

### 透明度（前端为主）
- `App.tsx` 岛容器当前 `blur ? "bg-card/70 backdrop-blur-xl" : "bg-card"`
- 改为按读到的 `window:opacity`（0.1~1.0）动态算背景色 alpha
  - 毛玻璃开：`backdrop-blur-xl` + 背景色 alpha 用 opacity 值（如 0.7）
  - 毛玻璃关：实色背景，opacity 仍作用于背景（实色 + 让点底透出来其实没意义，毛玻璃关闭时建议**忽略 opacity 强制 100%**，否则关了毛玻璃还半透明会看到桌面但没模糊，体验怪。**此项交用户在阶段 5 验证时确认行为**）
- 实现方式：Tailwind 任意 alpha 不支持运行时变量，用**内联 style `backgroundColor`**（从主题色变量取 hsl + 拼.alpha）。查 `App.tsx` 主题色来源（`globals.css` CSS 变量）后定写法
- 监听 `settings://changed` 的 `window:opacity` 即时重算，不动 motion.div 的 `animate opacity`（那是 hidden 态淡出动画，勿混）

### 配置导入导出
- **导出**：前端 save 对话框得路径 → `config_export(path)` → Rust 读三表组装 JSON 写盘
  - 导出文件名前端拼：`luckyisland-config-YYYYMMDD-HHmm.json`，时间戳前端生成传给 save 对话框默认名
  - 导出 `exportedAt` 也前端拼 ISO 字符串传 Rust（避免 Rust 侧时间）
  - settings 只导出可安全迁移的用户配置白名单；明确排除 `notify:http_token`、`ai:chat_api_key`、天气/股票缓存和 `ai:position` 等密钥或机器/运行时数据
- **导入**：前端 open 对话框得路径 → **前端弹确认对话框**「将覆盖所有设置、自选股与天气城市，确认？」→ 确认后 `config_import(path)`
  - Rust 侧：读文件 → `serde_json::from_str` → 校验 `version==1`（不符返错）→ **一个 SQLite 事务内**：只删除并重写可迁移 settings 白名单（密钥/token/缓存保留），全量覆盖 `stock_watchlist` 与 `weather_cities`（先清后写）
  - 事务后逐条 `app.emit("settings://changed", {key,value})`（前端各页重算）
  - 若导出含 `window:offset_x/y`，导入后额外调一次 `window_offset_apply` 让窗口上屏
  - 返回 `ImportSummary` 给前端显示覆盖条数
- **不动 `settings.rs`**：导入用 `db.setting_set`（storage 已有 public 方法），不走 `setting_set_and_emit`（那个是逐条命令，批量用直接 db 方法 + 手动 emit 更高效）

## 测试要点
- 透明度滑块拖动**即时生效**（毛玻璃开/关两种态分别测）

- 副屏断开回退主屏时偏移仍保留（`recover_island_to_monitor` 传 offsets）
- 重启后 opacity/offset 持久化恢复
- 导出 JSON 结构含 `version`/`exportedAt`/`settings`/`stockWatchlist`/`weatherCities`，肉眼可读可手编
- 导入：覆盖确认对话框弹出 → 导入后主题/页面开关/自选股/天气城市全部恢复成配置文件内容
- 导入 `general:theme`/`pages:enabled` 后灵动岛即时切换（验证 `settings://changed` 广播链）
- 导入 `window:offset_*` 后窗口真的移动（非仅改数据）
