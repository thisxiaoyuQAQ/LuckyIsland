## 当前接口

```text
ai_chat(requestId, provider, message, history)
  -> { reply, action, providerUsed }
ai_cancel(requestId)
  -> cancelled | already_finished | not_current
```

- Provider：Claude CLI、Codex CLI、自定义 OpenAI-compatible Chat API。
- `AiRuntime` 同时只登记一个 active request。
- `CancellationToken` 覆盖 CLI 子进程、HTTP、实时取证和 Chat API 工具轮次。
- AI 回答一次性完整返回；前端请求期间显示 pending/忙碌占位。当前没有后端 AI 分阶段进度事件。
- 语音另有 `voice://download-progress` 模型下载进度，不能与 AI 回答进度混称。

## Provider 与联网

- Claude CLI：`claude -p ... --output-format json`，允许自身 `WebSearch`，禁用 Bash/Edit/Write/NotebookEdit。
- Codex CLI：`codex exec`，prompt 走 stdin，最终消息从 `-o` 临时文件读取；`--ephemeral --sandbox read-only`，不使用 `--search`。
- Codex 实时问题：天气优先调用 uapis.cn，其他实时信息调用 DuckDuckGo HTML；证据非空后注入 prompt，查证失败时不调用 Codex。
- Chat API：调用 `{base_url}/chat/completions`；模型可请求后端 DuckDuckGo `web_search` function tool，400 时可降级为不带 tools 的请求。

“后端天气/DDG 预搜索”仅准确描述 Codex 路径；不能无差别套到 Claude CLI 或 Chat API。

## 语音

- sherpa-onnx KWS 检测唤醒词。
- cpal 采集并线性重采样到 16kHz。
- sherpa-onnx `OnlineRecognizer` 执行流式 ASR；唤醒后单轮转写并发出 `voice://transcript`。
- AI 面板支持 `voice_record_utterance` 按需录音。
- Windows SAPI5 播放唤醒应答。
- KWS/ASR 模型按需下载、检查、解压；存在 `voice://download-progress`。

Porcupine、Picovoice 与 AccessKey 是早期未采用方案，不是当前依赖或设置。
