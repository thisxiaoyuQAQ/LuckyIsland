## 接口与类型
```ts
// 前端 → Rust
invoke("ai_chat", { requestId, provider, message, history })
  → { reply, action, providerUsed }
invoke("ai_cancel", { requestId })
  → "cancelled" | "already_finished" | "not_current"
invoke("ai_switch_provider", { provider }) → void
invoke("ai_clear_history") → void
invoke("wake_set_enabled", { enabled }) → void

// Rust → 前端事件
"ai://wake"              // 唤醒词命中
"ai://action-result"     // { actionId, success, message }
"ai://provider-changed"  // { provider }
"voice://listening"      // boolean；按需录音开始 true，结束/失败 false
```
Rust：`AgentProvider` trait（CancellationToken 贯穿 CLI/HTTP/tool loop）；`AiRuntime` 维护 requestId 单活请求；`ActionRouter` 执行动作
配置字段：`[ai] [ai.claude_cli] [ai.codex_cli] [ai.chat_api] [ai.actions] [wake] [wake.asr] [wake.tts]`（见方案 §7）

## 实现要点（按里程碑）

### M7 AI 命令面板（核心）
- tauri.conf.json 新增 `ai-palette` 窗口（独立 label，Spotlight 风格，中央浮窗）
- `AgentProvider` trait：`async fn chat(history, system_prompt, cancellation_token) -> Result<String, ProviderError>`
- provider 实现：Claude CLI（保留 Child）/ Codex CLI（`codex --search exec`）/ Chat API（OpenAI 兼容 + DuckDuckGo function calling）
- system prompt 注入动作清单 + JSON schema + 用户当前快捷命令名（动态）
- 桌面动作统一用 prompt 约定 JSON；Chat API 仅为 `web_search` 使用原生 function calling
- `ActionRouter`：校验 args → 路由到 02/03/05/06 service → emit `ai://action-result`
- 全局热键 Alt+Space 唤起 AI 窗口（复用 global-shortcut 插件）
- ESC/失焦自动隐藏；对话历史持久化 SQLite `ai_conversations`
- provider 切换需 await，失败回滚；请求显式携带 provider，响应返回 `providerUsed`
- 真正取消与前端状态机按 [08a-AI请求取消.md](./08a-AI请求取消.md) 实施
- 安全：只允许预配置快捷命令名，不任意 spawn（防 prompt injection）

### M8 语音唤醒
- `cpal` 采集 16kHz 单声道 PCM
- `pv_porcupine` Rust binding 检测唤醒词（默认 "computer"）
- 命中 → emit `ai://wake` → 前端开 AI 窗口 + 灵动岛「正在听」波纹联动
- AccessKey 从 env / keyring 读；默认关闭，首次开启引导麦克风权限
- Porcupine 不可用时降级为热键呼出 + 通知

### M9 语音闭环
- 本地 ASR：sherpa-onnx 中文模型，VAD 静音 `vad_seconds` 结束录音
- 可选 TTS：Windows 系统 SAPI（免费本地）
- 唤醒 → 录音 → ASR → 文本填入 AI 输入框 → 自动发送（可配置）
- 按需录音 command 使用 async + `spawn_blocking`，`voice://listening` 在录音期间可被 WebView 实时处理

## 测试要点
- Alt+Space / 托盘 / 唤醒词 → AI 窗口出现；ESC/失焦 → 消失
- 切换 3 个 provider 均能对话（Claude/Codex CLI 需本机安装并登录）
- 「打开 codex 在 E:\Code」→ 灵动岛终端页新 tab，cwd 正确
- 「看股票」→ 跳股票页并展开；「加待办 买牛奶」→ 待办出现，可撤销
- 非法动作 / 解析失败 → 回退 reply，不崩溃
- 唤醒词 sensitivity 调节生效；关闭语音后麦克风流释放
- prompt injection（「跑 rm -rf」）→ 拒绝（不在快捷命令名内）
- 运行中终止 CLI/HTTP 后立即发下一轮，旧请求不得更新 UI、执行动作、写历史或 emit
- provider 切换失败回滚，天气等实时问题确认 Codex `--search` 与 Chat API 工具链实际生效
- 点击麦克风后「正在聆听」在录音期间可见，结束/失败后消失
