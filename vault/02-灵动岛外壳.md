# 02-灵动岛外壳

## 模块做啥（1 行）
透明无边框置顶窗口 + 系统托盘 + 两个可配置动作热键 + 多屏定位 + 主题 + 隐藏/紧凑/展开三态。

## 当前接口与设置

- 显示器：`monitor_list`、`monitor_get_selection`、`monitor_select`。
- 运行时监听：`monitor::start_runtime_watch(app)`，每 2 秒检查显示器集合。
- 事件：`monitor://changed`，向设置页广播可用列表与 fallback 状态。
- SQLite 设置：`window:monitor`、`window:offset_x`、`window:offset_y`、`window:opacity`。
- 可配置动作热键：`hotkeys:toggle_island`（默认 `alt+KeyX`）、`hotkeys:toggle_ai`（默认 `alt+Space`）；支持冲突检测、暂停、重新加载和恢复默认。

应用设置统一来自 `app_data_dir()/data.db` 的 `settings(key,value)`，不读取应用 TOML。

## 已实现

- Tauri 透明、无边框、置顶、跳过任务栏窗口；系统托盘与单实例。
- 岛窗口隐藏/紧凑/展开三态及主题/毛玻璃控制。
- 显示器选择、目标 DPI 物理居中、双轴偏移与边界 clamp。
- 选择具体显示器后立即移动，重启恢复选择。
- 已保存副屏离线时，应用主动恢复窗口尺寸、解除最小化并移到主屏，同时保留原副屏选择。
- 副屏恢复时后端只清除 fallback 状态并广播事件，**不主动跳回原屏**。

## 多屏产品边界

应用当前稳定承诺仅为“副屏断开时主动回退主屏”。2026-07-11 Windows 真机曾观察到副屏恢复后系统自动把窗口移回原屏，但这是 OS 行为观察，不是 LuckyIsland 的稳定保证。

点击穿透的最终产品契约由 DOC-10A-03 单独确认；本文不把早期 `setIgnoreCursorEvents` 设想描述为已实现能力，也不在 DOC-10A-01 中决定新行为。

早期规划中的 `window/mod.rs`、`transparent.rs`、`theme.rs`、`config/mod.rs`、Zustand stores 等不存在，均是历史未采用结构。

## 验证要点

- 两个动作热键修改、冲突检测、恢复默认。
- 副屏选择、立即移动、重启恢复。
- 副屏运行时断开主动回退且不覆盖保存选择。
- 副屏恢复时不把 OS 自动移动写成应用保证。
- 三态、托盘、主题和双轴偏移不回归。
