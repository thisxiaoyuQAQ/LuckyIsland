## 接口与类型
Rust → 前端事件：
- `window://state-changed`：`{ state: "hidden"|"compact"|"expanded" }`
- `theme://changed`：`{ mode: "light"|"dark" }`

前端 → Rust 命令：
- `set_island_state(state)`
- `get_monitors() -> Vec<MonitorInfo>`
- `set_monitor(name)`
- `set_click_through(enabled: bool)`

配置字段：`[window] [shortcut] [theme]`（见 docs/需求文档.md §7.1）

## 实现要点
- tauri.conf.json：`decorations:false, transparent:true, alwaysOnTop:true, skipTaskbar:true, resizable:false, shadow:false, visible:false`
- 启动后 Rust 读配置定位窗口、控制显示
- 点击穿透：`setIgnoreCursorEvents` + 前端 mouseover 检测进入岛
- 多屏：`availableMonitors()` 枚举，存 `name()`，启动匹配；找不到回退主屏
- 全局热键：`tauri-plugin-global-shortcut` 注册 Alt+X（可配置）
- 托盘：`tray-icon`，菜单 显示/隐藏·设置·重启·退出
- 主题：监听系统 + `data-theme` 属性 + CSS 变量；跟随/手动
- 单实例：`tauri-plugin-single-instance`

## 多屏选择与持久化增量（2026-07-11）

### 新增接口
- `monitor_list() -> MonitorInfo[]`
- `monitor_get_selection() -> MonitorSelectionState`
- `monitor_select(selection) -> MonitorSelectionState`
- `monitor::start_runtime_watch(app)`：运行时轮询显示器，副屏断开时临时回退主屏
- `monitor://changed` 事件：载荷 `MonitorSelectionState`，副屏断开/恢复时广播给设置页
- SQLite key：`window:monitor`，值为 `primary` 或具体显示器 ID。

### 已实现
- Rust `monitor` 模块统一处理显示器枚举、动态主屏、具体屏选择、目标 DPI 物理居中、立即移动和启动恢复。
- 设置页总体开关增加显示器选择，调用期间禁用，失败时保留旧选择并显示错误；监听 `monitor://changed` 实时更新回退提示与可用列表。
- 具体显示器缺失时临时回退主屏但保留选择。
- **运行时副屏断开回退**：2 秒轮询发现已保存的具体显示器离线时，立即把灵动岛临时移到主屏并 emit `monitor://changed(fallback=true)`；副屏恢复时不自动跳回窗口（需重启或手动重选），仅复位标志并 emit `fallback=false` 让设置页移除提示。F1.4「副屏恢复自动跳回」仍为 P1。

### 验证状态
- 自动化：显示器选择解析、缺失回退、未知选择拒绝、回退 ID、DPI 宽度和正/负坐标居中测试已覆盖（8 passed）。
- 真机：副屏即时移动、重启恢复、动态主屏语义、三态/热键回归均已通过；断开副屏重启回退已通过；运行时断开自动回退待真机确认。
