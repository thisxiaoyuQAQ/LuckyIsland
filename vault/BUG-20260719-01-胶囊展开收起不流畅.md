# BUG-20260719-01 修复：胶囊展开/收起不流畅

## 需要先读
- 项目备忘录.md
- docs/开发进度.md
- src/App.tsx（容器动画与 hover 提交链）
- src/lib/window-policy.ts（transition/hover controller）
- src-tauri/src/window_policy.rs（geometry_for / resize_window）

## Bug 摘要
- 实际表现：胶囊（240×80 原生窗口）↔ 展开/收起过渡分两段跳，不连贯。
- 期望表现：胶囊与面板之间宽度、高度同步平滑形变。
- 复现路径：开启「浮球」→ 岛休眠为胶囊 → 点击展开钮或悬停展开 → 再收起。
- 影响范围：胶囊态所有进出过渡（手动展开/收起、hover stage 2 展开与回落）。

## 根因假设
- 假设 1：宽度从不参与动画，只在原生层瞬时切换（`window_policy.rs` `resize_window` 一次 `set_size(240↔720)`），前端只动画 `height`（`App.tsx` 容器 `animate.height`）。
  - 证据：展开时原生 resize 先于高度动画提交（transitionController 对 expanded 无 wait），窗口瞬间变 720 宽细条；收起时先播完高度动画、240ms 延迟后窗口才收窄到 240 并横向重锚。
- 假设 2：`resize_window` 的 `set_size` 与 `set_position` 是两次独立系统调用，宽度变化时先缩后移，可能多抖一帧（次要，方案 A 不处理）。

## 选定方案
- 方案：A —— 前端 motion 容器同时动画 `width`（240↔内容宽）与 `height`，原生 resize 时机不变（展开在动画起点、收起在动画终点一次性执行），宽度跳变被前端动画遮住。
- 改动范围：`src/App.tsx` 容器动画参数与相位判断；纯前端，不动 Rust。
- 风险点：hover 展开/手动展开两路径需回归；收起阶段 width 动画必须与 collapseDelay（240ms）内的高度动画同步完成。
