## A. 高风险候选

### RISK-10C-01 PTY 生命周期

> 来源：AUD-RISK-01

验证自然 `exit`、显式 kill、重复 kill、reader 异常、应用退出、多 tab 压力下 registry/PID/缓存是否回收。先定义 dump 输出与幂等语义；确认问题后再设计 `TerminalSession` 和统一 finalize。不得以纯文件拆分替代生命周期测试。

### RISK-10C-02 语音下载与状态机

> 来源：AUD-RISK-02

验证模型下载取消/损坏/磁盘失败/重试、并发 start-stop-reload、双录音、设备断开与线程 panic。候选方案：RAII 下载 guard、异步落盘、`spawn_blocking` 解压、唯一 `.part`/临时目录、统一状态机。不得调整已验 ASR/KWS/VAD 参数，除非出现新证据。

### RISK-10C-03 Provider 拆分与协议边界

> 来源：AUD-REF-07、AUD-RISK-03

只有在现有取消/联网/编码/进程树测试完整保护后，才按 `claude_cli`、`codex_cli`、`chat_api`、`live_lookup` 渐进拆分。保持 `AgentProvider`、stdin、120 秒总超时、A+ 证据门禁和 requestId 生命周期。不重开 BUG-02/04/05。

### RISK-10C-04 AI 动作原子性与幂等

> 来源：AUD-RISK-04

先决定：副作用提交前/后取消语义、历史成对写入、重复请求幂等、可逆动作返回 ID。候选方案为强类型 action、事务、request/action ID 和窄范围补偿；无产品语义前不大改 router。

### RISK-10C-05 command 权限与 CSP

> 来源：AUD-RISK-05

先建立窗口 × command × settings key 的允许/拒绝矩阵，并验证 Tauri capability 的真实边界；优先给敏感 command 做调用窗口校验和 key 白名单，再分阶段收紧 CSP。静态审计尚未证明可利用漏洞，不以“已存在漏洞”表述。

### RISK-10C-06 Composition root、错误和平台边界

> 来源：AUD-RISK-06

在补足窗口状态/错误传播/后台 shutdown 测试后，再将 `lib.rs` 收敛为 composition root；用枚举替代状态字符串；Windows-only 依赖和实现集中管理。产品当前以 Windows 为主，不要求跨平台重写。

## B. 产品待决

### PROD-10C-01 完整待办范围

> 来源：AUD-PROD-01

选项：A. 0.2.x 补齐截止、优先级、编辑、排序、提醒；B. 当前定位为轻量待办并下调 PRD；C. 分阶段先编辑/排序，提醒后置。**推荐 C**，利用后端已有字段，避免一次性加入调度提醒。

### PROD-10C-02 导航与专注模式

> 来源：AUD-PROD-02

分别决定：启动默认页还是记住上次页；是否实现 Tab；是否提供切页方案设置；专注模式是否属于当前版本。不要把四项绑定成一个大功能。

### PROD-10C-03 通知与窗口可配置项

> 来源：AUD-PROD-03

决定通知是否可禁止打断、是否需要声音、窗口尺寸是否开放。推荐先做“通知打断”单开关；声音/尺寸后置，避免扩大测试矩阵。

### PROD-10C-04 AI 历史、撤销与确认

> 来源：AUD-PROD-04

推荐先实现默认 30 天过期清理；若需要撤销，仅让 `add_todo` 返回 ID 并支持窄范围删除；三个以上可逆动作出现前不建通用 Undo Bus。动作前确认维持默认关，危险动作另行分类。

### PROD-10C-05 点击穿透语义

> 来源：AUD-PROD-05
> 状态：✅ 已决策（2026-07-13）

用户确认 Hidden 与鼠标穿透分离：Hidden 继续调用 `window.hide()`；穿透是默认关闭、可持久化且可选绑定全局热键的整窗手动开关，设置页始终可恢复。不做透明区域选择性 hit-test 或系统级鼠标靠近自动解锁。实现并入模块 11，RUN-10C-03 保留用于 Windows 真机点击矩阵；本决策本身不代表穿透已实现。

## 派生任务门禁

只有满足以下条件才创建独立 BUG/重构 Vault：

1. 有与旧 BUG 不同的具体输入/状态和错误结果；
2. 有复现证据或用户接受的产品决策；
3. 写明与 BUG-20260710-01～05/08a 的区别；
4. 有自动化与真机验收边界；
5. 不与 10a/10b 已有任务重复。

## 明确不做

- 无证据的大规模重写；
- 跨平台重构；
- 统一三种 Provider 的底层传输；
- 全页面无条件 keep-alive；
- 立即启用极严 CSP 而无逐窗口回归；
- 用新状态框架替代 SQLite KV/现有 Rust IO 边界。
