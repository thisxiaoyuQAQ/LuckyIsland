- `test:rust`；

## B. 渐进重构

### REF-10B-01 统一 Tauri event 生命周期（优先）

> 来源：AUD-REF-01

建立 `useTauriEvent`/`useAsyncSubscription`：卸载前完成则正常清理，卸载后才 resolve 则立即 unlisten；handler 用 ref 保持最新；统一 Promise rejection。先测试，再按 `App`、Stock、Notify、Weather、Terminal 分批迁移。

### REF-10B-02 共享主题同步

> 来源：AUD-REF-02

提取 `parseThemeMode`、系统主题读取、DOM 应用和 `startThemeSync(): dispose`；React 提供薄 hook。保持 render 前应用主题，防止首帧闪烁；复验岛/设置/AI 三窗口同步。

### REF-10B-03 轻量拆分 App

> 来源：AUD-REF-03
> 依赖：REF-10B-01、页面 registry 契约测试

只提取页面 registry、`useIslandSettings`、`useIslandEvents` 和导航；保留根窗口状态。Terminal 等重资源页面是否保活单独评估，不让所有隐藏页面永久运行定时器/监听器。

### REF-10B-04 SQLite 版本化 migration

> 来源：AUD-REF-04

采用 `PRAGMA user_version` 或 `schema_migrations`；当前 schema 为基线；每版事务执行。测试空库、旧库、重复执行、失败回滚。物理 SQL 目录次要；不顺便引入 ORM/连接池。因涉及用户数据，实施时独立批次与备份策略。

### REF-10B-05 持久、脱敏、轮转日志

> 来源：AUD-REF-05

覆盖启动/退出、Provider 开始取消超时失败、CLI 退出、HTTP 绑定、PTY、显示器回退、语音模型失败。禁止 API key、token、完整 prompt/对话。Release 下可定位日志，设置大小/天数轮转；UI 错误仍需可理解。

### REF-10B-06 后续渐进项

> 来源：AUD-REF-06

- IPC 原始响应按 unknown 校验，先 AI 历史/通知/天气/股票；
- Weather 按城市管理 pending/error/generation 与请求去重；
- AiPalette 先用 reducer/state machine 测试取消/晚到响应，再拆 Provider/语音/UI；
- 设置字段分离 draft/persisted，按字段 debounce/blur，并显示保存错误。

这些是 P2，不一次性全量迁移，也不为了类型美观引入大型代码生成体系。
