# 01-基础脚手架

## 模块做啥（1 行）
搭建 Tauri 2 + React + TS + Tailwind v4 + shadcn/ui 可运行的空灵动岛窗口骨架。

## 依赖谁（1 行）
- 必须先完成：无（首个模块）
- 可并行：无

## 需要先读哪几个文件
- 项目备忘录.md（必读）
- docs/技术栈规划.md「§2 前端技术栈」「§7 开发环境」
- docs/需求文档.md「§3.1 灵动岛容器」「§3.8 主题」

## 接口与类型
本模块无对外接口，仅产出可运行骨架。约定：
- 包管理器：pnpm
- Tauri 窗口 label：`island`
- 前端入口：`src/main.tsx` → `App.tsx`
- 样式入口：`src/styles/globals.css`（`@import "tailwindcss";`）
- 主题：`data-theme` 属性 + CSS 变量

## 实现要点
- 用 `create-tauri-app` 生成 react-ts 模板（identifier `com.luckyisland.app`）
- Tailwind v4：装 `tailwindcss @tailwindcss/vite`，Vite 插件模式，**无需** tailwind.config.js
- shadcn/ui：`shadcn init`（Tailwind v4 模式），先 add：button / switch / dialog
- 主题变量：globals.css 定义 `:root` 与 `[data-theme="dark"]` 两套 CSS 变量
- tauri.conf.json 窗口暂用普通可见配置（M1 再改透明无边框），先保证 dev 跑通
- 保留并补全项目根 `.gitignore`
- 验证：`pnpm dev` 前端起来；`pnpm tauri dev` 弹出空白窗口

## 测试要点
- `pnpm dev` 无报错，页面显示占位
- `pnpm tauri dev` 能弹出窗口（首次 cargo 编译较慢，可接受）
- `pnpm build` 前端能产出 dist/

## 产出清单
